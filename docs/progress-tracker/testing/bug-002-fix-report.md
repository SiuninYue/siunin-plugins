## 🐛 Bug 描述

**症状**: `batch_scanner` 在多 pattern 输入时可能出现重复文件，且输出顺序不稳定。

**影响**:
- 结果集可重复，降低下游处理质量
- 输出顺序波动导致测试脆弱、难以比对

---

## 🔍 根本原因

1. 聚合文件列表时缺少显式去重逻辑。
2. 输出结果未施加稳定排序约束。
3. 相关测试断言过弱，未及时暴露问题。

---

## ✅ 解决方案

### 1. 增加去重与稳定排序

- 引入 `seen` 集合去重。
- 输出前按 `path` 排序，保证确定性。

### 2. 强化测试覆盖

- 新增重复移除测试。
- 新增排序稳定性测试。
- 修复弱断言（`>= 0`）为更有约束力的断言。
