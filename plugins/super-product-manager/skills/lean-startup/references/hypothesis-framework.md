# 假设验证框架

## 三种核心假设

| 假设类型 | 核心问题 | 风险等级 |
|---------|---------|---------|
| **问题假设** | 这个问题真的存在吗？ | ⭐⭐⭐ 最高 |
| **解决方案假设** | 我的方案能解决问题吗？ | ⭐⭐ 高 |
| **商业假设** | 用户愿意付钱吗？ | ⭐ 中 |

**验证顺序**：先验证最危险的假设

风险排序：
1. 用户根本不需要这个（问题不存在）
2. 用户不会用你的方案（解决方案不对）
3. 用户不愿意付钱（商业模式不成立）

---

## 假设类型详解

### 1. 问题假设

**核心问题**：这个问题真的存在吗？

| 验证方法 | 说明 | 成本 |
|---------|------|------|
| 用户访谈 | 直接问用户 | 低 |
| 搜索量分析 | 看有多少人在搜索 | 低 |
| 竞品分析 | 看有没有人在解决 | 低 |
| 论坛/社区 | 看用户在讨论什么 | 低 |
| Landing Page | 看有没有人感兴趣 | 低 |

**验证标准**：
- ✅ 多个用户主动提到这个问题
- ✅ 用户为解决这个问题付出过努力
- ✅ 现有解决方案让用户不满意

**假设模板**：
```markdown
## 问题假设卡片

**假设内容**：
[目标用户] 在 [特定场景] 下会遇到 [问题]，
这个问题 [频率] 发生，
造成的影响是 [损失/痛苦]。

**验证方法**：
1. 访谈 X 个目标用户
2. 问：你上次遇到这个问题是什么时候？

**成功标准**：
- X/X 用户确认有这个问题
- X/X 用户尝试过解决
```

---

### 2. 解决方案假设

**核心问题**：我的方案能解决问题吗？

| 验证方法 | 说明 | 成本 |
|---------|------|------|
| 原型测试 | 做个原型让用户试 | 中 |
| 人工模拟 | 人工提供服务 | 中 |
| 小范围试用 | 找早期用户使用 | 中 |
| 视频演示 | 演示产品概念 | 低 |

**验证标准**：
- ✅ 用户能完成核心任务
- ✅ 用户体验到价值
- ✅ 用户愿意继续使用

**假设模板**：
```markdown
## 解决方案假设卡片

**假设内容**：
通过 [解决方案]，
[目标用户] 可以 [获得的好处]，
比现有方案 [更好的地方]。

**验证方法**：
1. 做出 MVP
2. 让 X 个用户使用
3. 观察 [行为指标]

**成功标准**：
- 激活率 > X%
- 留存率 > X%
- 用户反馈满意度 > X
```

---

### 3. 商业假设

**核心问题**：用户愿意付钱吗？

| 验证方法 | 说明 | 成本 |
|---------|------|------|
| 预售测试 | 先收钱再做 | 低 |
| 定价测试 | A/B 测试不同价格 | 中 |
| 付费墙测试 | 看有多少人愿意付费 | 中 |
| 众筹 | 真金白银预购 | 低 |

**验证标准**：
- ✅ 有用户真的付费
- ✅ 付费率达到可接受水平
- ✅ LTV > CAC

**假设模板**：
```markdown
## 商业假设卡片

**假设内容**：
[目标用户] 愿意为 [价值主张] 付费，
可接受的价格范围是 [价格区间]，
付费方式偏好是 [订阅/买断/按次]。

**验证方法**：
1. 设置付费功能
2. 测试不同价格点
3. 观察付费转化率

**成功标准**：
- 付费转化率 > X%
- 客单价 > ¥X
- LTV/CAC > 3
```

## 假设优先级矩阵

### 评估维度

| 维度 | 问题 |
|-----|------|
| **风险** | 如果假设错了，后果多严重？ |
| **不确定性** | 我们对这个假设有多少把握？ |
| **验证成本** | 验证这个假设需要多少资源？ |

### 优先级矩阵

```
            高不确定性              低不确定性
         ┌─────────────────────────────────────┐
         │                                     │
高风险   │   ⭐⭐⭐ 最优先            ⭐ 观察   │
         │   立即验证                可能已验证 │
         │                                     │
         ├─────────────────────────────────────┤
         │                                     │
低风险   │   ⭐⭐ 其次              不需要验证 │
         │   时间允许时验证         可以假设成立│
         │                                     │
         └─────────────────────────────────────┘
```

### 假设清单模板

```markdown
## 假设清单

| 假设 | 类型 | 风险 | 不确定性 | 优先级 | 验证状态 |
|-----|------|------|---------|-------|---------|
| [假设1] | 问题 | 高 | 高 | ⭐⭐⭐ | 待验证 |
| [假设2] | 方案 | 高 | 中 | ⭐⭐ | 验证中 |
| [假设3] | 商业 | 中 | 高 | ⭐⭐ | 待验证 |
| [假设4] | 问题 | 低 | 低 | - | 已验证 |
```

## 验证方法对照表

| 假设类型 | 推荐方法 | 样本量 | 周期 |
|---------|---------|-------|------|
| 问题假设 | 用户访谈 | 5-10人 | 1-2周 |
| 问题假设 | Landing Page | 500+访客 | 1周 |
| 方案假设 | 原型测试 | 10-20人 | 2-4周 |
| 方案假设 | MVP 试用 | 50-100人 | 4-8周 |
| 商业假设 | 预售测试 | 100+人 | 2-4周 |
| 商业假设 | A/B 定价 | 1000+人 | 2-4周 |

## 常见陷阱

| 陷阱 | 问题 | 解决 |
|-----|------|------|
| **假设太模糊** | 无法验证 | 写具体、可测量 |
| **同时验证太多** | 资源分散 | 聚焦最高优先级 |
| **验证太久** | 错过时机 | 设定时间限制 |
| **确认偏差** | 只看支持的证据 | 主动找反面证据 |
| **样本太小** | 结论不可靠 | 达到最小样本量 |
| **不记录假设** | 忘记初心 | 写下来、跟踪 |
