# 精益画布 (Lean Canvas)

## 画布概览

一页纸说清楚你的商业模式：

```
┌────────────────┬─────────────────┬────────────────┐
│    问题 (2)    │   解决方案 (4)   │   独特卖点 (3) │
│                │                 │                │
│ 用户最痛的     │ 你的解决方案     │ 为什么选你     │
│ 3 个问题       │ Top 3 功能      │ 不选别人       │
│                │                 │                │
├────────────────┼─────────────────┼────────────────┤
│   关键指标 (8)  │                 │   不公平优势   │
│                │                 │     (9)        │
│ 衡量成功的     │   价值主张 (1)   │                │
│ 关键数据       │                 │ 别人抄不走的   │
│                │ 一句话说清楚    │                │
├────────────────┤ 为什么值得用    ├────────────────┤
│    渠道 (5)    │                 │   用户群体 (1) │
│                │                 │                │
│ 怎么触达用户   │                 │ 你的目标用户   │
│                │                 │ 是谁           │
├────────────────┴─────────────────┴────────────────┤
│           成本结构 (7)     │     收入来源 (6)       │
│                           │                        │
│ 主要成本有哪些            │ 怎么赚钱               │
│                           │                        │
└───────────────────────────┴────────────────────────┘
```

**填写顺序**：按括号里的数字 1-9

---

## 各模块详解

### 1. 用户群体 (Customer Segments)

**问题**：你的目标用户是谁？

**填写要求**：
- 具体描述，不要"所有人"
- 如果有多个细分，列出优先级
- 区分用户和客户（付费的人）

**模板**：
```markdown
## 用户群体

**主要用户**：
[具体描述：年龄、职业、场景]

**用户画像**：
- 人口特征：[年龄、地区、收入]
- 行为特征：[习惯、偏好]
- 痛点特征：[遇到什么问题]

**早期采用者**：
[最先会用你产品的那批人是谁]
```

---

### 2. 问题 (Problem)

**问题**：用户最痛的问题是什么？

**填写要求**：
- 列出 Top 3 痛点
- 要是真实存在的问题
- 写出现有替代方案

**模板**：
```markdown
## 问题

**痛点 Top 3**：
1. [最痛的问题]
2. [第二痛的问题]
3. [第三痛的问题]

**现有替代方案**：
- [用户现在怎么解决]
- [这些方案的问题是什么]
```

---

### 3. 独特卖点 (Unique Value Proposition)

**问题**：为什么用户选你不选别人？

**填写要求**：
- 一句话说清楚
- 清晰、具体、可信
- 不是功能列表，是价值承诺

**公式**：
```
为 [目标用户]
解决 [核心问题]
通过 [解决方案]
比 [替代方案] 更 [差异化优势]
```

**好的 UVP 示例**：
- Uber：一键叫车，几分钟到达
- Slack：让团队沟通更高效
- Airbnb：像当地人一样旅行

---

### 4. 解决方案 (Solution)

**问题**：你怎么解决用户的问题？

**填写要求**：
- 列出 Top 3 核心功能
- 对应痛点 Top 3
- 写功能，不写技术

**模板**：
```markdown
## 解决方案

| 痛点 | 解决方案 |
|-----|---------|
| [痛点1] | [功能1] |
| [痛点2] | [功能2] |
| [痛点3] | [功能3] |
```

---

### 5. 渠道 (Channels)

**问题**：怎么触达用户？

**填写要求**：
- 列出获客渠道
- 区分免费和付费渠道
- 考虑早期和规模化阶段

**模板**：
```markdown
## 渠道

**早期渠道**（前 1000 用户）：
- [渠道1]：[策略]
- [渠道2]：[策略]

**规模化渠道**：
- [渠道1]：预计 CAC ¥X
- [渠道2]：预计 CAC ¥X
```

---

### 6. 收入来源 (Revenue Streams)

**问题**：怎么赚钱？

**填写要求**：
- 写清楚收费模式
- 写出定价策略
- 估算关键数据

**模板**：
```markdown
## 收入来源

**收费模式**：
订阅 / 买断 / 按次 / 免费增值 / 广告

**定价**：
| 版本 | 价格 | 功能 |
|-----|------|------|
| 免费版 | ¥0 | [功能] |
| 付费版 | ¥X/月 | [功能] |

**关键假设**：
- 付费转化率：X%
- ARPU：¥X
- LTV：¥X
```

---

### 7. 成本结构 (Cost Structure)

**问题**：主要成本是什么？

**填写要求**：
- 固定成本 vs 变动成本
- 早期和规模化的成本差异
- 估算盈亏平衡点

**模板**：
```markdown
## 成本结构

**固定成本**（每月）：
- 人力：¥X
- 办公：¥X
- 工具：¥X

**变动成本**（每用户）：
- 服务器：¥X
- 获客：¥X
- 支持：¥X

**盈亏平衡**：
需要 X 个付费用户
```

---

### 8. 关键指标 (Key Metrics)

**问题**：怎么衡量成功？

**填写要求**：
- 参考 AARRR 漏斗
- 选择最重要的 3-5 个
- 包括北极星指标

**模板**：
```markdown
## 关键指标

**北极星指标**：[指标名]

**AARRR 指标**：
| 阶段 | 指标 | 目标 |
|-----|------|------|
| 获取 | [指标] | X |
| 激活 | [指标] | X% |
| 留存 | [指标] | X% |
| 变现 | [指标] | X% |
| 推荐 | [指标] | X |
```

---

### 9. 不公平优势 (Unfair Advantage)

**问题**：你有什么别人抄不走的？

**填写要求**：
- 诚实评估，不自欺欺人
- 如果没有，承认"暂无"
- 这是要持续建立的

**什么是不公平优势**：
| 有效 | 无效 |
|-----|------|
| 内部团队信息 | 功能（可以抄）|
| 个人权威 | 技术（可以学）|
| 社区 | 想法（谁都能想）|
| 现有客户 | 先发优势（可追赶）|
| 专利/版权 | 价格（可以降）|
| 网络效应 | |

## 使用建议

### 画布的用途

| 场景 | 怎么用 |
|-----|-------|
| 梳理想法 | 逼自己想清楚每个模块 |
| 团队对齐 | 确保大家理解一致 |
| 对外沟通 | 给投资人/合作伙伴看 |
| 定期回顾 | 验证假设，更新画布 |

### 迭代频率

- 早期：每 2-4 周更新
- 成长期：每月更新
- 成熟期：每季度回顾

### 常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| 填太满 | 什么都想做 | 聚焦核心 |
| 太抽象 | 没有具体信息 | 写具体、可验证 |
| 不更新 | 画完就放着 | 定期回顾更新 |
| 只是形式 | 填了但没用 | 指导实际决策 |
