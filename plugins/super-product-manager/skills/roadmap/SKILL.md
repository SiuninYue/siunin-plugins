## 核心说明

产品路线图技能。用于阶段目标、里程碑与资源规划。

---

## 路线图类型识别

| 类型 | 适用场景 | 特点 | 时间跨度 |
|-----|---------|------|---------|
| **Now-Next-Later** | 敏捷团队、快速变化 | 三段式，灵活 | 季度更新 |
| **Theme-Based** | 战略导向、Outcome驱动 | 按主题组织，不承诺具体功能 | 半年/年度 |
| **Feature-Based** | 功能明确、需求稳定 | 按功能里程碑组织 | 季度/半年度 |
| **Outcome-Based** | 增长导向、数据驱动 | 按业务目标组织 | 季度/半年度 |

### 快速选择

```
用户输入
    │
    ├── 敏捷团队？需求经常变？
    │   └── 是 → Now-Next-Later
    │
    ├── 关注业务目标，具体功能待定？
    │   └── 是 → Theme-Based
    │
    ├── 功能需求已明确？
    │   └── 是 → Feature-Based
    │
    └── 有明确的数据指标？
        └── → Outcome-Based
```

## Now-Next-Later 路线图

### 结构

| 阶段 | 说明 | 时间 | 颗粒度 |
|-----|-----|-----|-------|
| **Now** | 正在做或即将做 | 当前迭代/季度 | 具体功能 |
| **Next** | 接下来要做 | 下个迭代/季度 | 主题或功能 |
| **Later** | 未来可能做 | 以后 | 主题方向 |

### 示例

```markdown
# 产品路线图：Q1 2024

## Now (当前迭代)
- [ ] 用户认证系统
- [ ] 商品浏览功能
- [ ] 购物车功能

## Next (下个迭代)
- 支付系统对接
- 订单管理
- 用户中心

## Later (未来方向)
- 会员体系
- 营销工具
- 数据分析
```

## 里程碑规划

### 里程碑定义

每个里程碑必须包含：
- **目标**：要达成什么
- **交付物**：具体功能或成果
- **验收标准**：如何验证完成
- **负责人**：谁负责
- **时间**：什么时候完成
- **依赖**：依赖什么

### 里程碑模板

```markdown
## 里程碑 M1：用户系统

### 目标
用户可以注册、登录、管理个人信息

### 交付物
- [ ] 注册功能（手机验证码）
- [ ] 登录功能
- [ ] 个人中心页面
- [ ] 密码找回功能

### 验收标准
- 注册成功率 > 95%
- 登录响应时间 < 1秒
- 通过安全测试

### 负责人：张三
### 时间：Week 1-2
### 依赖：短信服务开通
```

| 设计稿 | 内部 | UI开发 | 先做后端接口 |

## 致命问题检查

路线图常见错误：

- [ ] **过度承诺**：排期太满，没有缓冲
- [ ] **没有优先级**：什么都重要等于什么都不重要
- [ ] **一成不变**：市场变化了也不调整
- [ ] **没有资源考虑**：假设资源无限
- [ ] **只有没有Next和Later**：只做当前，不考虑未来
- [ ] **没有里程碑**：无法追踪进度

**记住**：路线图不是承诺，是规划。必须根据实际情况调整。

## 示例对话

### 用户说："帮我规划产品路线图"

**输出**：
> 规划路线图之前，先回答：
>
> 1. **产品愿景是什么？** 最终想做成什么样？
> 2. **规划多长时间？** 一个月？一个季度？一年？
> 3. **有多少资源？** 几个人？每周能投入多少时间？
> 4. **现在什么阶段？** 还没开始？已经有用户了？
>
> 不知道这些，路线图就是在画饼。

>

## 常见错误对照

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **排期太满** | 没有缓冲，延期必崩 | 预留20%缓冲时间 |
| **没有优先级** | P0和P2混在一起 | 明确Now/Next/Later |
| **一成不变** | 不根据实际情况调整 | 每月回顾调整 |
| **只有Now** | 不考虑后续规划 | 至少规划3个月 |
| **没有验收** | 无法判断是否完成 | 每个里程碑有验收标准 |

## 最佳实践

### 1. 从愿景到具体

先有产品愿景，再拆成目标，最后才是具体功能。

### 2. 优先级清晰

Now做最重要的，Next做次重要的，Later是未来可能做的。

### 3. 定期回顾

每个月回顾路线图，根据实际情况调整。

### 4. 沟通透明

让团队知道路线图是什么，为什么这样规划。

### 5. 现实一点

不要画大饼，基于实际资源和能力规划。
