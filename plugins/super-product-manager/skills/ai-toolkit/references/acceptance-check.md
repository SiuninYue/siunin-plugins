# Acceptance Checklist 生成模板

Acceptance Checklist 让 AI 在交付前自检，减少幻觉和遗漏。

---

## Acceptance Checklist 模板

```markdown
## Acceptance Checklist: [任务名称]

### 交付前自检清单

请在提交前逐项检查，确保全部满足：

---

### 1. 必须做到（Must Have）

- [ ] **[检查项1]**
  - 标准：[具体判断标准]
  - 示例：[合格示例]

- [ ] **[检查项2]**
  - 标准：[具体判断标准]
  - 示例：[合格示例]

---

### 2. 禁止出现（Must Not）

- [ ] **不能 [禁止行为1]**
  - 原因：[为什么禁止]
  - 错误示例：[不合格示例]

- [ ] **不能 [禁止行为2]**
  - 原因：[为什么禁止]
  - 错误示例：[不合格示例]

---

### 3. 边界情况覆盖

- [ ] **空数据**：当数据为空时，[期望行为]
- [ ] **极值**：当数据达到极限时，[期望行为]
- [ ] **错误输入**：当输入错误时，[期望行为]
- [ ] **并发**：当同时操作时，[期望行为]
- [ ] **网络异常**：当网络断开时，[期望行为]

---

### 4. 一致性检查

- [ ] 术语与 Context Pack 中的定义一致
- [ ] 格式与 Output Spec 中的要求一致
- [ ] 不包含 Context Pack 中标注的"待确认假设"

---

### 5. 完整性检查

- [ ] 所有必须项都已完成
- [ ] 没有遗漏的边界情况
- [ ] 没有未解决的 TODO/TBD

---

### 自检结果

| 类别 | 通过 | 未通过 | 说明 |
|------|------|--------|------|
| Must Have | /[总数] | | |
| Must Not | /[总数] | | |
| 边界覆盖 | /[总数] | | |
| 一致性 | /[总数] | | |
| 完整性 | /[总数] | | |

**总体评估**：[ ] 可交付 / [ ] 需修改
```

## 常见检查项

### 通用检查项

```markdown
### Must Have
- [ ] 响应了所有输入的问题
- [ ] 使用了正确的格式
- [ ] 信息准确无误
- [ ] 逻辑自洽无矛盾

### Must Not
- [ ] 不能编造不存在的信息
- [ ] 不能忽略约束条件
- [ ] 不能超出任务范围
- [ ] 不能使用模糊表述（如"可能""大概"）
```

### PRD 检查项

```markdown
### Must Have
- [ ] 包含背景和目标
- [ ] 包含用户故事
- [ ] 包含功能范围（In/Out Scope）
- [ ] 包含验收标准
- [ ] 包含边界情况处理

### Must Not
- [ ] 不能有模糊的需求描述
- [ ] 不能缺少异常处理
- [ ] 不能使用"待定/TBD"而不标注
```

### 代码检查项

```markdown
### Must Have
- [ ] 代码可运行无语法错误
- [ ] 包含必要的注释
- [ ] 处理了边界情况
- [ ] 包含错误处理

### Must Not
- [ ] 不能有硬编码的敏感信息
- [ ] 不能有未使用的变量
- [ ] 不能有明显的性能问题
```

### 设计检查项

```markdown
### Must Have
- [ ] 描述了所有页面状态
- [ ] 描述了交互流程
- [ ] 考虑了响应式设计
- [ ] 包含空状态/错误状态

### Must Not
- [ ] 不能遗漏关键交互
- [ ] 不能忽略异常状态
- [ ] 不能使用模糊描述
```

## 示例

```markdown
## Acceptance Checklist: 评论功能 PRD

### 1. Must Have
- [ ] **包含评论发表流程**
  - 标准：描述从输入到显示的完整流程
  - 示例：用户输入 → 点击发送 → 敏感词检查 → 显示评论

- [ ] **包含回复功能**
  - 标准：描述回复的触发和展示方式
  - 示例：点击回复按钮 → 弹出输入框 → 回复显示在父评论下

### 2. Must Not
- [ ] **不能缺少权限说明**
  - 原因：需要明确谁可以评论
  - 错误示例：没有说明未登录用户的行为

### 3. 边界覆盖
- [ ] **空评论**：按钮禁用，不提交
- [ ] **超长评论**：显示字数限制提示
- [ ] **敏感词**：阻止发送，提示修改
- [ ] **网络错误**：保留内容，提示重试

### 自检结果
| 类别 | 通过 | 说明 |
|------|------|------|
| Must Have | 5/5 | |
| Must Not | 2/2 | |
| 边界覆盖 | 4/4 | |
| 一致性 | 3/3 | |

**总体评估**：[x] 可交付
```

## 生成要点

### 可检查性
- 每个检查项必须是可以明确判断 Yes/No 的
- 避免"做得好不好"这种主观判断

### 边界情况
- 边界情况是 AI 最容易遗漏的
- 提前列出常见边界情况

### 自检 vs 他检
- 这个清单主要给 AI 自检用
- 人工检查时也可以用这个清单
