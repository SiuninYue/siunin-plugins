# MoSCoW 分类法

**按必要性分类**，特别适合 MVP 规划和范围控制。

---

## 四个类别

| 类别 | 含义 | 比例建议 |
|-----|------|---------|
| **M**ust Have | 必须有，没有不能上线 | 60% |
| **S**hould Have | 应该有，上线后尽快补上 | 20% |
| **C**ould Have | 可以有，有余力就做 | 15% |
| **W**on't Have | 不做，明确放弃 | 5% |

---

## 判断标准

### Must Have（必须有）
**没有它产品不能用**

✅ 满足以下任一条件：
- 核心用户流程必需
- 法律/合规要求
- 安全相关
- 没有它产品无法运行

❌ 常见误区：
- "我觉得很重要" ≠ Must Have
- "竞品都有" ≠ Must Have
- "老板说要" ≠ Must Have

### Should Have（应该有）
**重要但有替代方案**

✅ 满足以下条件：
- 影响体验但不致命
- 大部分用户会用
- 有临时替代方案
- 上线后可以尽快补上

### Could Have（可以有）
**锦上添花**

✅ 满足以下条件：
- 只有部分用户需要
- 没有也不影响核心流程
- 竞品有但不是差异化点
- 做了更好，不做也行

### Won't Have（不做）
**明确放弃**

✅ 满足以下任一条件：
- 投入产出比太低
- 与产品方向不符
- 技术上不可行
- 超出当前资源能力

**重要**：Won't Have 不是"以后再做"，是"这个版本明确不做"。

## MoSCoW 分类模板

```markdown
## 功能分类表：[产品/版本名称]

### Must Have（必须有）- 60%
- [ ] 用户登录/注册
- [ ] 核心功能 A
- [ ] 核心功能 B
- [ ] 安全相关功能

### Should Have（应该有）- 20%
- [ ] 功能 C：优化体验
- [ ] 功能 D：提升效率

### Could Have（可以有）- 15%
- [ ] 功能 E：锦上添花
- [ ] 功能 F：部分用户需要

### Won't Have（不做）- 明确放弃
- 功能 G（原因：超出当前资源）
- 功能 H（原因：与方向不符）
- 功能 I（原因：ROI 太低）
```

---

## 分类流程

### Step 1：列出所有功能
把要做的功能全部列出来，不要遗漏。

### Step 2：先分 Must Have
问自己：**"没有这个功能，产品能上线吗？"**
- 能上线 → 不是 Must Have
- 不能上线 → Must Have

### Step 3：再分 Won't Have
问自己：**"这个功能值得做吗？"**
- 不值得 → Won't Have
- 值得但资源不够 → Won't Have（这个版本）

### Step 4：剩下的分 Should/Could
- 大部分用户需要 → Should Have
- 部分用户需要 → Could Have

### Step 5：检查比例
- Must Have 超过 60%？→ 重新审视，可能标准太松
- Won't Have 是空的？→ 不敢说不，这是危险信号

## 示例：评论功能 MVP

```markdown
## 功能分类表：评论功能 V1

### Must Have（必须有）
- [x] 发表评论（纯文本，限 500 字）
- [x] 查看评论列表
- [x] 删除自己的评论
- [x] 敏感词过滤

### Should Have（应该有）
- [ ] 回复他人评论（一级）
- [ ] 评论分页加载

### Could Have（可以有）
- [ ] 评论点赞
- [ ] 评论排序（按时间/热度）

### Won't Have（不做）
- 评论图片/表情（原因：增加审核成本）
- 多级嵌套回复（原因：复杂度高，用户量不确定）
- 评论举报（原因：需要人工审核，暂无资源）
```

## 常见问题

**Q: 怎么说服老板某个功能是 Could Have 不是 Must Have？**
A: 问他"没有这个功能，产品能上线吗？"，如果能，就不是 Must Have。

**Q: Won't Have 和"以后再做"有什么区别？**
A: Won't Have 是明确说"这个版本不做"，"以后再做"应该放在 Could Have。

**Q: Should Have 做不完怎么办？**
A: 砍掉一部分变成 Could Have，保证 Must Have 全部完成。

**Q: 分类之后发现 Must Have 太多做不完？**
A: 要么延期，要么重新审视哪些是真的 Must Have。
