# 2x2 优先级矩阵

**可视化决策工具**，适合快速分类和团队沟通。

---

## 矩阵图

```
影响大 │
       │   ┌─────────────┬─────────────┐
       │   │             │             │
       │   │   优先做     │   马上做     │
       │   │  Quick Win  │  重点项目   │
       │   │ (高影响低成本)│ (高影响高成本)│
       │   │             │             │
───────┼───┼─────────────┼─────────────┤
       │   │             │             │
       │   │   不做       │   谨慎考虑   │
       │   │  别浪费时间  │  ROI 存疑   │
       │   │ (低影响低成本)│ (低影响高成本)│
       │   │             │             │
影响小 │   └─────────────┴─────────────┘
        成本低 ──────────────────── 成本高
```

---

## 四个象限的策略

| 象限 | 特征 | 策略 | 优先级 |
|-----|------|------|-------|
| **右上** | 高影响 + 高成本 | 重点项目，仔细规划 | 🔴 高 |
| **左上** | 高影响 + 低成本 | 立刻做，Quick Win | 🔴 最高 |
| **左下** | 低影响 + 低成本 | 有空再做，或自动化 | 🟡 低 |
| **右下** | 低影响 + 高成本 | 不做，浪费资源 | ⚫ 不做 |

---

## 判断维度

### 影响大 vs 影响小
| 影响大 | 影响小 |
|-------|-------|
| 核心指标相关 | 边缘指标 |
| 大部分用户受影响 | 少数用户受影响 |
| 解决主要痛点 | 解决次要问题 |
| 战略价值高 | 战术价值 |

### 成本高 vs 成本低
| 成本高 | 成本低 |
|-------|-------|
| 需要多人多周 | 1-2 人几天 |
| 涉及架构改动 | 局部修改 |
| 需要跨团队协作 | 单团队完成 |
| 技术风险高 | 技术风险低 |

## 分类流程

### Step 1：评估影响
对每个功能问：
- 影响多少用户？
- 影响核心指标吗？
- 解决的问题有多痛？

### Step 2：评估成本
对每个功能问：
- 需要多少人？
- 需要多长时间？
- 技术风险高吗？

### Step 3：放入象限
根据评估结果放入对应象限。

### Step 4：行动
- 左上（Quick Win）→ 立刻安排
- 右上（重点项目）→ 排入 Roadmap
- 左下（低优先）→ Backlog
- 右下（不做）→ 明确放弃

## 常见问题

**Q: 功能很难判断是高影响还是低影响怎么办？**
A: 用数据说话。如果没有数据，先做 A/B 测试验证。

**Q: 团队对影响/成本判断有分歧怎么办？**
A: 让技术评估成本，让产品/数据评估影响，减少主观判断。

**Q: 右上象限（重点项目）太多做不完？**
A: 进一步用 RICE 评分细化排序。
