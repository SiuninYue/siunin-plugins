# 技术规格指南

## 延迟加载规则

- 默认只使用本文件内容
- 只有在用户明确要求细节或需要模板时，才加载 references 中的附件
- 未加载附件时，先给摘要与下一步

## 核心原则

**产品经理不需要会写代码，但需要能和开发说清楚技术需求**。技术规格的本质是「双方对齐」。

## 技术规格文档结构

### 标准模板

```markdown
## 技术规格：[功能名称]

### 文档信息
| 项目 | 内容 |
|-----|-----|
| 版本 | v1.0 |
| 日期 | [日期] |
| 产品经理 | [姓名] |
| 技术负责人 | [姓名] |

---

### 1. 功能概述
[一句话描述功能目的]

### 2. 数据结构
[核心数据模型]

### 3. 接口设计
[API 列表和规格]

### 4. 业务逻辑
[核心流程和规则]

### 5. 非功能需求
[性能、安全、可用性]

### 6. 技术风险
[潜在问题和应对]
```

## 产品经理必备技术概念

### 前后端分工

| 层级 | 职责 | 产品关注点 |
|------|------|-----------|
| **前端** | 界面展示、用户交互 | 交互体验、加载速度 |
| **后端** | 业务逻辑、数据处理 | 功能逻辑、数据准确 |
| **数据库** | 数据存储 | 数据结构、查询效率 |

### 常见技术术语

| 术语 | 含义 | 产品关心什么 |
|------|------|-------------|
| **API** | 前后端通信接口 | 需要哪些接口 |
| **SDK** | 封装好的工具包 | 接入第三方能力 |
| **缓存** | 加速数据读取 | 数据时效性 |
| **异步** | 后台处理 | 用户等待体验 |
| **分页** | 分批加载数据 | 列表加载效率 |
| **限流** | 控制请求频率 | 系统稳定性 |

### 常见性能指标

| 指标 | 含义 | 一般目标 |
|------|------|---------|
| **响应时间** | 请求到响应耗时 | < 500ms |
| **可用性** | 系统正常运行时间 | > 99.9% |
| **并发数** | 同时处理请求数 | 看业务规模 |
| **吞吐量** | 单位时间处理量 | 看业务需求 |

## PRD 协作引导

本技能可补充 PRD 中的以下章节（仅当 PRD 已包含该章节时）：

| 可补充章节 | 补充内容 | 输出位置 |
|-----------|---------|---------|

| **依赖与风险**（技术部分） | 技术依赖、技术风险、应对措施 | `08-dependencies.md` 或 PRD 对应章节 |

### 协作约束

- ✅ 只补充上述已有章节，不新增章节
- ✅ 技术规格需包含验收标准（可测试）
- ❌ 不编辑其他章节（如用户故事、功能需求等）
