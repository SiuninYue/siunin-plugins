# 北极星指标

## 什么是北极星指标

**北极星指标 = 最能代表产品核心价值的单一指标**

就像北极星指引航向，北极星指标是整个团队的共同目标。

---

## 经典案例

| 公司 | 北极星指标 | 为什么是它 |
|-----|----------|----------|
| **Facebook** | 日活用户数 (DAU) | 社交网络价值在于连接人 |
| **Airbnb** | 预订间夜数 | 完成预订才有价值 |
| **Slack** | 团队发送消息数 | 沟通是核心价值 |
| **Netflix** | 观看时长 | 内容消费是核心 |
| **Uber** | 完成行程数 | 出行服务的核心 |
| **Spotify** | 收听时长 | 音乐消费的核心 |
| **Dropbox** | 活跃共享用户数 | 协作是核心价值 |

## 选择北极星指标的标准

### 好的北极星指标具备

| 特征 | 说明 |
|-----|------|
| **代表核心价值** | 用户从这个行为中获得价值 |
| **可衡量** | 能准确、持续地测量 |
| **可行动** | 知道怎么提升它 |
| **领先指标** | 能预测未来业务表现 |
| **团队共识** | 各部门都认可 |

### 北极星指标不应该是

| 不好的指标 | 问题 |
|----------|------|
| **收入** | 滞后指标，知道收入下降时已经晚了 |
| **注册用户数** | 不代表价值交付 |
| **页面浏览量** | 不代表用户获得价值 |
| **NPS** | 太抽象，难以行动 |

## 北极星指标选择流程

### Step 1：明确产品核心价值

问自己：**用户从产品中获得什么价值？**

| 产品类型 | 核心价值 |
|---------|---------|
| 社交产品 | 连接与沟通 |
| 内容产品 | 消费与娱乐 |
| 工具产品 | 效率提升 |
| 交易产品 | 完成交易 |
| SaaS 产品 | 工作产出 |

### Step 2：找到价值交付的关键行为

问自己：**用户做什么行为时获得这个价值？**

```markdown
核心价值：[价值描述]

关键行为选项：
- 行为 A：[描述]
- 行为 B：[描述]
- 行为 C：[描述]

最能代表价值交付的行为：[选择]
```

### Step 3：量化关键行为

问自己：**这个行为怎么量化？**

| 行为 | 量化方式 |
|-----|---------|
| 用户沟通 | 发送消息数、活跃对话数 |
| 内容消费 | 观看时长、完成率 |
| 完成任务 | 任务完成数、效率提升 |
| 交易 | 成交订单数、GMV |

### Step 4：验证指标

| 验证问题 | 要求 |
|---------|------|
| 它和收入相关吗？ | 指标增长应带动收入增长 |
| 它和留存相关吗？ | 指标高的用户留存应更好 |
| 各部门能影响它吗？ | 产品、运营、市场都能贡献 |
| 能持续追踪吗？ | 数据采集稳定可靠 |

## 北极星指标分解

### 输入指标树

```
           北极星指标
               │
    ┌──────────┼──────────┐
    │          │          │
  输入1      输入2      输入3
    │          │          │
┌───┴───┐  ┌───┴───┐  ┌───┴───┐
│       │  │       │  │       │
细分1  细分2 细分1 细分2 细分1 细分2
```

### 示例：电商平台

```
           GMV (北极星)
               │
    ┌──────────┼──────────┐
    │          │          │
  访客数     转化率      客单价
    │          │          │
┌───┴───┐  ┌───┴───┐  ┌───┴───┐
│       │  │       │  │       │
自然   付费  浏览   加购  件数   单价
流量   流量  →加购  →支付
```

### 分解的作用

| 层级 | 负责人 | 作用 |
|-----|-------|------|
| 北极星 | 全公司 | 统一目标 |
| 一级输入 | 部门 | 明确职责 |
| 二级细分 | 团队 | 具体行动 |

## 常见问题

### Q1：可以有多个北极星吗？

**原则上只有一个**。多个「北极星」会导致团队目标分散。

如果确实有多个重要指标，可以：
- 选一个作为主北极星
- 其他作为「护栏指标」（不能变差的指标）

### Q2：北极星需要改变吗？

**需要定期审视**，但不要频繁更换。

| 场景 | 是否需要改变 |
|-----|-------------|
| 业务阶段变化 | 可能需要（如从增长到盈利） |
| 季度波动 | 不需要 |
| 发现更好的指标 | 需要慎重评估 |

### Q3：北极星和 OKR 的关系

```
北极星指标 = 公司级 Key Result
     ↓
部门 OKR 围绕北极星展开
     ↓
个人 OKR 支撑部门 OKR
```

## 北极星指标常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **选了虚荣指标** | 注册用户数不代表价值 | 选择代表价值交付的指标 |
| **太复杂** | 团队理解不了 | 简单、清晰、可解释 |
| **只看北极星** | 忽视护栏指标 | 同时关注质量指标 |
| **频繁更换** | 团队无所适从 | 保持稳定 |
| **各做各的** | 部门目标和北极星脱节 | 确保分解对齐 |
