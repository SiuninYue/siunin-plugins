## 各阶段详解

### 1. Acquisition 获取

**核心问题**：用户从哪来？来了多少？

#### 关键指标

| 指标 | 定义 | 健康范围 |
|-----|------|---------|
| **新增用户数** | 每日/周/月新注册用户 | 持续增长 |
| **获客成本 (CAC)** | 获取一个用户的成本 | 低于 LTV 的 1/3 |
| **渠道转化率** | 从曝光到注册的转化 | 因渠道而异 |
| **渠道质量** | 各渠道用户的后续表现 | 高留存渠道优先 |

```markdown
| 付费广告 | X | ¥X | X% | ¥X | X |
| 社交媒体 | X | ¥X | X% | ¥X | X |
| 推荐 | X | ¥X | X% | ¥X | X |

**最优渠道**：[渠道名] - ROI 最高
**待优化渠道**：[渠道名] - CAC 过高
```

### 2. Activation 激活

**核心问题**：用户体验到产品核心价值了吗？

#### 定义激活

| 产品类型 | 激活定义示例 |
|---------|-------------|
| 社交产品 | 添加 X 个好友 |
| 工具产品 | 完成首次核心操作 |
| 内容产品 | 观看 X 分钟内容 |
| 电商产品 | 完成首次购买 |
| SaaS 产品 | 完成核心功能使用 |

#### 关键指标

| 指标 | 定义 | 健康范围 |
|-----|------|---------|
| **激活率** | 完成激活的用户比例 | 30-50% |
| **激活时间** | 从注册到激活的时间 | 越短越好 |
| **首次体验完成率** | 新手引导完成率 | 70%+ |

#### 激活漏斗分析

```markdown

| 步骤 | 用户数 | 转化率 | 流失原因 |
|-----|-------|-------|---------|
| 注册完成 | 1000 | 100% | - |
| 进入产品 | 900 | 90% | 注册后未打开 |
| 完成引导 | 700 | 78% | 引导太长/太复杂 |
| 首次核心操作 | 500 | 71% | 不知道怎么用 |
| **激活** | 400 | **40%** | - |

**最大流失点**：[步骤] - 转化率只有 X%
**优化方向**：[具体措施]
```

### 3. Retention 留存

**核心问题**：用户会回来吗？

#### 留存类型

| 类型 | 定义 | 用途 |
|-----|------|------|
| **次日留存** | D1 回访比例 | 评估首次体验 |
| **周留存** | D7 回访比例 | 评估核心价值 |
| **月留存** | D30 回访比例 | 评估长期粘性 |

#### 健康留存基准

| 产品类型 | 次日留存 | 周留存 | 月留存 |
|---------|---------|-------|-------|
| 社交产品 | 40-50% | 25-35% | 15-25% |
| 游戏 | 35-45% | 15-25% | 5-15% |
| 工具产品 | 30-40% | 20-30% | 10-20% |
| 电商 | 15-25% | 10-15% | 5-10% |

### 4. Revenue 变现

**核心问题**：用户愿意付钱吗？

#### 关键指标

| 指标 | 定义 | 说明 |
|-----|------|------|
| **付费转化率** | 付费用户 / 活跃用户 | 评估变现效率 |
| **ARPU** | 平均每用户收入 | 用户价值 |
| **ARPPU** | 平均每付费用户收入 | 付费用户价值 |
| **LTV** | 用户生命周期价值 | 长期价值 |
| **LTV/CAC** | 生命周期价值比获客成本 | 应 > 3 |

#### LTV 计算

```
LTV = ARPU × 用户生命周期

用户生命周期 ≈ 1 / 月流失率

示例：
- 月 ARPU = ¥50
- 月流失率 = 10%
- 生命周期 = 1 / 0.1 = 10 个月
- LTV = ¥50 × 10 = ¥500
```

#### 变现漏斗

```markdown
## 变现漏斗

| 步骤 | 用户数 | 转化率 |
|-----|-------|-------|
| 活跃用户 | 10,000 | 100% |
| 浏览付费页 | 3,000 | 30% |
| 点击购买 | 1,000 | 33% |
| 完成支付 | 500 | 50% |
| **付费率** | - | **5%** |

**流失原因分析**：
- 未浏览付费页：不知道有付费功能
- 未点击购买：价格/价值不匹配
- 未完成支付：支付流程问题
```

### 5. Referral 推荐

**核心问题**：用户愿意推荐吗？

#### 关键指标

| 指标 | 定义 | 健康范围 |
|-----|------|---------|
| **K因子** | 每个用户带来的新用户数 | > 1 = 病毒增长 |
| **推荐率** | 发起推荐的用户比例 | 因场景而异 |
| **推荐转化率** | 推荐链接的转化率 | 10-30% |
| **NPS** | 净推荐值 | > 50 优秀 |

#### K 因子计算

```
K = 发起推荐的用户比例 × 每人发送邀请数 × 邀请转化率

示例：
- 20% 用户发起推荐
- 平均发送 5 个邀请
- 邀请转化率 10%
- K = 0.2 × 5 × 0.1 = 0.1

K > 1：病毒增长
K < 1：需要配合其他获客渠道
```

## AARRR 优化优先级

### 一般优先级

```
1. 留存 (Retention) - 留不住用户，其他都是浪费
2. 激活 (Activation) - 没激活就不会留存
3. 获取 (Acquisition) - 确保有稳定用户来源
4. 变现 (Revenue) - 商业化
5. 推荐 (Referral) - 增长加速器
```

### 阶段性重点

| 阶段 | 重点 | 说明 |
|-----|------|------|
| 早期 | 激活 + 留存 | 验证产品价值 |
| 增长期 | 获取 + 留存 | 规模化增长 |
| 成熟期 | 变现 + 留存 | 商业化 |
