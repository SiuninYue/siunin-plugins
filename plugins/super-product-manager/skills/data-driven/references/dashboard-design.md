# 数据看板设计

## 看板层级体系

| 层级 | 受众 | 更新频率 | 核心内容 |
|-----|------|---------|---------|
| **战略看板** | CEO/高管 | 周/月 | 北极星指标、关键业务指标 |
| **运营看板** | 运营团队 | 日 | AARRR 漏斗、渠道数据 |
| **产品看板** | 产品团队 | 日 | 功能使用、用户行为 |
| **技术看板** | 开发团队 | 实时 | 性能、错误率、可用性 |

---

## 看板设计原则

### 核心原则

| 原则 | 说明 | 示例 |
|-----|------|------|
| **一眼看懂** | 5秒内能看到最重要的信息 | 大数字、颜色区分 |
| **有对比** | 同比、环比、目标对比 | +15% vs 上周 |
| **能下钻** | 从汇总钻取到细节 | 点击查看详情 |
| **有告警** | 异常数据自动提醒 | 红黄绿灯 |
| **不要太多** | 一个看板最多 6-8 个指标 | 聚焦核心 |

### 视觉层级

```
┌─────────────────────────────────────────────────┐
│                                                  │
│    ████  核心指标 (最大、最显眼)                  │
│                                                  │
├──────────────────────┬──────────────────────────┤
│                      │                          │
│   ██ 次要指标1       │   ██ 次要指标2           │
│                      │                          │
├──────────────────────┴──────────────────────────┤
│                                                  │
│   █ 趋势图 / 详细数据                            │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 数据告警设置

### 告警级别

| 级别 | 定义 | 响应时间 | 通知方式 |
|-----|------|---------|---------|
| 🔴 紧急 | 核心功能不可用 | 立即 | 电话 + 短信 |
| 🟠 重要 | 核心指标异常 | 1小时内 | 即时消息 |
| 🟡 警告 | 指标接近阈值 | 当天 | 邮件 |
| 🟢 正常 | 一切正常 | - | 不通知 |

### 告警规则示例

```markdown
## 告警规则配置

### 业务告警

| 规则 | 条件 | 级别 | 接收人 |
|-----|------|------|-------|
| 订单下降 | 实时订单量 < 昨日同期 50% | 🟠 | 运营组 |
| 支付失败 | 支付失败率 > 5% | 🔴 | 技术组 + 运营组 |
| 新增骤降 | 新增用户 < 日均 30% | 🟡 | 增长组 |

### 技术告警

| 规则 | 条件 | 级别 | 接收人 |
|-----|------|------|-------|
| 服务不可用 | 可用性 < 99% | 🔴 | 技术组全员 |
| 响应变慢 | P99 > 1000ms | 🟠 | 值班人员 |
| 错误激增 | 错误率 > 1% | 🟠 | 值班人员 |
```

## 看板维护

### 定期审视

| 频率 | 检查内容 |
|-----|---------|
| 每周 | 指标是否还在关注？ |
| 每月 | 是否需要新增/删除指标？ |
| 每季 | 看板结构是否需要调整？ |

### 常见问题

| 问题 | 症状 | 解决 |
|-----|------|------|
| **指标太多** | 看板杂乱，找不到重点 | 聚焦 5-8 个核心指标 |
| **指标过时** | 没人看，不做决策 | 删除或替换 |
| **数据不准** | 指标定义混乱 | 统一定义，验证埋点 |
| **更新太慢** | 看的是昨天的数据 | 优化数据管道 |
