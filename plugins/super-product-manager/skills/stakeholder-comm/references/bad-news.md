# 坏消息传达

## 传达原则

| 原则 | 说明 |
|-----|------|
| **快** | 第一时间通报，不要等 |
| **全** | 说清楚全部影响，不要隐瞒 |
| **有方案** | 不只是报告问题，要有解决方案 |
| **有改进** | 说明怎么防止再次发生 |
| **负责任** | 不推卸，承担该承担的 |

---

## 传达技巧

### 时机选择

| 情况 | 时机 |
|-----|------|
| 紧急问题 | 立即，不要等 |
| 重要问题 | 当天内 |
| 一般问题 | 下次例会或周报 |

### 渠道选择

| 紧急程度 | 渠道 |
|---------|------|
| 紧急 | 电话 + 即时消息 |
| 重要 | 即时消息 + 邮件 |
| 一般 | 邮件 / 周报 |

### 措辞技巧

| 避免 | 改为 |
|-----|------|
| "这不是我的错" | "发生了问题，原因是..." |
| "我们没办法" | "我们正在尝试...方案" |
| "你们应该..." | "我建议我们可以..." |
| "这是历史遗留问题" | "原因是...，我们计划..." |

---

## 常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **拖延不报** | 小问题变大问题 | 第一时间通报 |
| **隐瞒影响** | 信任受损 | 透明说明全部影响 |
| **推卸责任** | 解决不了问题 | 先解决问题再追责 |
| **只报问题** | 让人焦虑 | 问题+方案一起说 |
| **过度承诺** | 更大的失望 | 实事求是 |
