# 向上汇报 (Executive Summary)

## 汇报原则

| 原则 | 说明 |
|-----|------|
| **结论先行** | 先说结论，再说过程 |
| **数据说话** | 少说感觉，多给数据 |
| **问题要有方案** | 别只说问题，要有解决方案 |
| **时间要短** | 5分钟内说完核心 |
| **准备好被追问** | 细节放在附录 |

---

## 汇报技巧

### 金字塔原理

```
           结论
            │
    ┌───────┼───────┐
    │       │       │
  论据1   论据2   论据3
    │       │       │
   细节    细节    细节
```

**先说结论**，再给支撑论据，细节放最后。

### SCQA 框架

| 要素 | 说明 | 示例 |
|-----|------|------|
| **S**ituation | 背景情况 | "我们上季度用户增长 30%" |
| **C**omplication | 问题挑战 | "但留存率下降到 15%" |
| **Q**uestion | 核心问题 | "如何提升留存？" |
| **A**nswer | 解决方案 | "我们计划从三个方面入手..." |

### 数据可视化原则

| 场景 | 推荐图表 |
|-----|---------|
| 趋势变化 | 折线图 |
| 对比分析 | 柱状图 |
| 占比构成 | 饼图/堆叠图 |
| 关系分布 | 散点图 |
| 进度展示 | 进度条 |

## 常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **过程太多** | 领导没时间听你怎么做的 | 结论先行 |
| **没有数据** | 只有感觉没有依据 | 用数据说话 |
| **只报问题** | 让人觉得你只会抱怨 | 问题+方案 |
| **太长太细** | 领导没时间 | 控制在 5 分钟 |
| **报喜不报忧** | 最后一起暴雷 | 及时透明 |
| **不准备追问** | 被问住了 | 准备 FAQ |

---

## 被追问时的回应

| 情况 | 回应方式 |
|-----|---------|
| 知道答案 | 直接回答，简洁清晰 |
| 不知道但能查 | "这个数据我会后会确认，X 小时内回复您" |
| 需要时间分析 | "这个问题需要更深入分析，我 X 天内给您详细报告" |
| 超出能力范围 | "这个需要和 [谁] 确认，我来协调" |
