# 项目状态更新

## 状态定义

| 状态 | 图标 | 定义 | 行动 |
|-----|------|------|------|
| 🟢 绿灯 | 正常 | 按计划进行 | 继续 |
| 🟡 黄灯 | 有风险 | 可能延期，需要关注 | 密切关注，准备应对 |
| 🔴 红灯 | 有问题 | 已经出问题，需要介入 | 立即处理，升级 |

## 状态更新频率

| 项目阶段 | 更新频率 | 更新内容 |
|---------|---------|---------|
| 启动期 | 每周 | 准备情况、风险识别 |
| 执行期 | 每周 | 进度、问题、下周计划 |
| 关键期 | 每日 | 关键任务状态 |
| 收尾期 | 每周 | 验收情况、遗留问题 |
| 紧急状态 | 即时 | 问题状态、处理进展 |

---

## 状态更新原则

| 原则 | 说明 |
|-----|------|
| **及时性** | 问题要及时上报，不要等到无法挽回 |
| **准确性** | 数据要准确，状态要真实 |
| **简洁性** | 突出重点，不要事无巨细 |
| **可追溯** | 有记录，可以回溯历史 |
| **差异化** | 不同受众不同内容深度 |

---

## 抄送策略

| 受众 | 内容深度 | 频率 |
|-----|---------|------|
| **直属领导** | 详细，包括问题和风险 | 每周 |
| **高层领导** | 摘要，结论先行 | 双周/月度 |
| **相关方** | 与其相关的内容 | 按需 |
| **团队成员** | 全部细节 | 每日/每周 |
