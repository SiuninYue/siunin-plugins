# Journey Trace 9 阶段追问清单

按需加载。仅在需要每阶段详细追问时读取本文件。

---

## 阶段 1：Discovery（发现）

开发者第一次听说这个工具/API/框架。

- 搜索相关关键词，第一页结果中有没有你的产品？
- GitHub/npm/PyPI 页面的描述能在 10 秒内说清楚"这是什么/解决什么问题"吗？
- 有没有一句话 pitch："X 是一个 Y，用于 Z"？
- 与竞品相比的差异化是否清晰？

---

## 阶段 2：First Contact（第一接触）

打开文档或 README 的第一屏。

- README 第一段能否在 30 秒内告诉开发者：能做什么 / 适合谁 / 怎么开始？
- 有没有可以直接复制的代码片段（不需要滚动就能看到）？
- 版本信息是否清晰标注？（避免读了旧文档浪费时间）
- 有没有 "⚠️ 仅适用于 X 环境" 等前置声明？

---

## 阶段 3：Setup（环境搭建）

安装、配置、环境准备。

- 安装命令是否 ≤ 1 行？（`npm install X` / `pip install X`）
- 有无需要手动创建配置文件？能否提供 init 命令自动生成？
- 环境变量是否有 `.env.example` 模板？
- 依赖冲突是否有明确说明（需要 Node ≥ X，Python ≥ Y）？
- 安装失败的常见原因是否有文档？

---

## 阶段 4：Hello World（第一个可运行结果）

运行第一个可工作的示例。

- 从安装完成到看到第一个输出需要几步操作？
- 每一步是否都有预期输出说明？（"你应该看到…"）
- 第一个示例是否足够简单（< 10 行代码）？
- 示例是否能直接复制运行，无需任何修改？
- TTHW 实测时间（Champion < 2min / Competitive 2-5min / Needs Work 5-10min / Red Flag >10min）

---

## 阶段 5：Core Task（核心任务）

完成一个真实的、非玩具的任务。

- 文档中是否有"如何实现 [常见真实场景]"的指引？
- 从 Hello World 到核心任务的跨越是否太大？（学习曲线断层）
- 是否有完整的端到端示例（不只是片段）？
- 最常用的 API 是否有参数说明 + 返回值说明 + 错误码说明？
- 示例代码是否有注释解释"为什么这么写"？

---

## 阶段 6：Error Recovery（错误恢复）

遇到第一个错误，尝试解决。

- 错误信息是否告诉开发者：发生了什么 / 为什么发生 / 怎么修？
- 常见错误码是否有文档对应？
- 错误堆栈是否有足够上下文（不只是"Error: undefined"）？
- 是否有 troubleshooting 页面收录高频问题？
- 社区（GitHub Issues / Discord / StackOverflow）是否活跃？搜索能找到相关问题的答案吗？

---

## 阶段 7：Advanced Usage（高级用法）

使用非主路径功能、扩展点、高级配置。

- 高级配置选项是否有完整文档（不只是 TypeScript 类型定义）？
- 是否有插件/扩展机制？如何开发？
- 如何覆盖默认行为（escape hatch）？
- 有无性能调优指南？
- 是否有 API 参考文档（不只是 guide 式教程）？

---

## 阶段 8：Collaboration（协作）

多人协作、权限管理、代码审查流程。

- 配置文件是否适合提交到 Git？（有无敏感信息）
- 是否有 CI/CD 集成指南？
- 多人使用时是否有配置冲突问题？
- 版本锁定（lock file）策略是否清晰？
- code review 时需要注意的特殊模式是否有文档？

---

## 阶段 9：Maintenance（长期维护）

升级依赖、迁移、长期维护。

- CHANGELOG 是否清晰标注 breaking changes？
- 是否有版本迁移指南（migration guide）？
- 是否有 LTS / deprecation 策略？
- 安全漏洞如何报告和追踪？
- 项目是否在活跃维护中？（最近一次 commit 是什么时候？）
