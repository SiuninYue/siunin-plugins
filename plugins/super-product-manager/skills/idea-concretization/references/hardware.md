# 硬件/IoT

## 核心价值主张

- **用户价值**：为目标用户提供[独特功能/性能提升/体验改善]，在[使用场景]下解决[具体痛点]，相比软件方案有[硬件独特优势]
- **商业价值**：通过[硬件销售/服务订阅/数据增值]模式，实现[毛利率目标]和[市场规模]，差异化在于[技术创新/工业设计/用户体验]
- **验证指标**：产品良率 > [X]%，用户满意度 > [Y]%，售后率 < [Z]%

## 追加问题

1. **产品定义**：硬件解决了什么软件无法解决的问题？核心功能和性能指标是什么？
2. **工业设计**：外观、材质、人机交互如何设计？用户体验测试结果如何？
3. **技术实现**：关键技术难点和解决方案？核心元器件供应链是否稳定？
4. **制造与供应链**：生产工厂选择标准？最小起订量、生产周期、良率目标？
5. **合规与认证**：需要哪些安全、电磁兼容、无线电等认证？认证周期和成本？
6. **成本与定价**：BOM成本、制造成本、物流成本、售后成本？定价策略和毛利空间？
7. **售后支持**：保修政策、维修网络、备件管理、软件更新策略？

## 关键成功因素

- **产品定义准确**：精准把握用户需求，硬件解决方案有明显优势
- **技术实现可靠**：核心技术创新可行，供应链稳定，质量可控
- **工业设计优秀**：外观美观，用户体验好，制造工艺成熟
- **成本控制有效**：BOM成本优化，生产良率高，物流效率高
- **生态建设完整**：软件配套、数据服务、售后支持形成完整体验

## 常见陷阱与规避

- **技术风险低估**：关键技术未经验证 → 先做原型验证，选择成熟技术方案
- **供应链风险**：核心元器件供应不稳定 → 多供应商策略，建立安全库存
- **成本控制失效**：BOM成本超预期 → 早期与供应链合作，优化设计方案
- **认证周期延误**：合规认证时间低估 → 早期咨询认证机构，预留缓冲时间
- **售后成本过高**：维修率超预期 → 加强质量测试，优化维修流程

## 建议文档（按需）

- **产品需求文档**：详细的功能规格、性能指标、用户体验要求
- **技术方案文档**：系统架构、关键技术方案、元器件选型、测试计划
- **工业设计规范**：外观设计、材质选择、人机交互、包装设计
- **供应链管理计划**：供应商清单、采购流程、质量控制、库存管理
- **制造与生产计划**：工厂选择、生产流程、测试标准、良率目标
- **合规认证清单**：所需认证类型、测试标准、时间计划、预算
- **售后支持体系**：保修政策、维修网络、备件管理、用户支持

## 验证优先级指导

- **P0（必须验证）**：用户对硬件解决方案的需求、核心技术可行性、成本结构合理性
- **P1（重要验证）**：供应链稳定性、制造可行性、合规认证要求
- **P2（后续验证）**：规模化生产成本、售后支持体系、软件生态建设
