# SaaS/B2B

## 核心价值主张

- **用户价值**：为目标企业客户解决[具体业务问题/效率痛点]，在[行业/场景]下提供[功能/数据/自动化]解决方案，节省时间/降低成本/提升效果
- **商业价值**：通过[订阅制/按用量/按功能模块]收费，实现[客户生命周期价值]和[收入增长]，差异化在于[行业专精/集成能力/数据智能]
- **验证指标**：客户获取成本 < [X]元，客户生命周期价值 > [Y]元，净收入留存率 > [Z]%

## 追加问题

1. **目标客户与痛点**：具体服务哪些行业/规模的企业？他们最迫切的业务问题是什么？
2. **决策链与采购**：购买决策涉及哪些角色（用户、主管、IT、财务）？预算来源和审批流程？
3. **产品价值量化**：如何量化给客户带来的价值（节省时间、降低成本、增加收入）？
4. **集成与迁移**：需要与哪些现有系统集成？数据迁移成本和风险？
5. **安全与合规**：需要满足哪些安全标准（SOC2、ISO27001）和行业合规要求？
6. **多租户架构**：如何设计数据隔离、性能扩展、定制化能力？
7. **客户成功**：如何确保客户持续使用并获得价值？续约、增购、口碑机制？

## 关键成功因素

- **行业理解深度**：深刻理解目标行业的业务流程和痛点
- **产品价值清晰**：能明确量化给客户带来的商业价值
- **销售流程有效**：匹配企业采购决策流程，降低销售周期
- **客户成功体系**：确保客户上线、使用、获得价值、续约增购
- **技术架构稳健**：安全、可靠、可扩展，支持企业级需求

## 常见陷阱与规避

- **产品过于通用**：试图服务所有行业 → 聚焦垂直行业，做深不做广
- **销售周期过长**：未能匹配企业采购流程 → 设计标准销售流程和材料
- **客户成功缺失**：只关注签单忽略使用 → 建立客户成功团队和流程
- **技术债务积累**：为快速上线牺牲架构 → 坚持企业级技术标准
- **定价策略不当**：未能反映价值或超出客户预算 → 做价值导向定价测试

## 建议文档（按需）

- **产品需求文档**：功能规格、用户场景、验收标准、发布计划
- **技术架构文档**：多租户设计、安全架构、集成方案、扩展规划
- **销售材料**：客户案例、价值计算器、竞品对比、提案模板
- **安全与合规文档**：安全白皮书、合规清单、数据保护方案
- **客户成功计划**：客户onboarding流程、培训材料、成功指标、续约管理
- **定价策略文档**：定价模型、套餐设计、折扣政策、合同模板

## 验证优先级指导

- **P0（必须验证）**：目标客户真实痛点、产品价值认可度、定价接受度
- **P1（重要验证）**：销售流程可行性、集成技术要求、安全合规门槛
- **P2（后续验证）**：客户成功体系有效性、规模化销售能力、生态建设
