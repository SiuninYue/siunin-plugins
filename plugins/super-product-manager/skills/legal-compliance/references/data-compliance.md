# 数据合规指南

## 核心法规

### 国内主要法规

| 法规 | 适用范围 | 核心要求 |
|------|---------|---------|
| **个人信息保护法** | 处理个人信息 | 告知同意、最小必要 |
| **网络安全法** | 网络运营者 | 等级保护、安全义务 |
| **数据安全法** | 数据处理活动 | 分类分级、安全评估 |
| **儿童个人信息规定** | 涉及14岁以下 | 监护人同意 |

### 关键合规要求

| 要求 | 说明 |
|------|------|
| **告知同意** | 收集前告知用途，获得同意 |
| **最小必要** | 只收集业务必需的信息 |
| **目的限制** | 不得超范围使用 |
| **安全保护** | 技术和管理措施保护 |
| **权利保障** | 响应用户查询、更正、删除请求 |

## 数据分类分级

### 分类标准

| 类别 | 示例 | 保护要求 |
|------|------|---------|
| **一般数据** | 昵称、设置偏好 | 基本保护 |
| **个人信息** | 手机号、邮箱 | 告知同意 |
| **敏感个人信息** | 身份证、银行卡、生物信息 | 单独同意、加密存储 |
| **核心数据** | 关系国家安全 | 最高保护级别 |

### 敏感个人信息清单

```markdown
## 敏感个人信息识别

### 需要单独同意的信息
- 身份证号码
- 银行卡号
- 生物识别信息（人脸、指纹）
- 医疗健康信息
- 金融账户信息
- 行踪轨迹
- 14岁以下未成年人信息

### 处理要求
1. 单独弹窗告知
2. 用户主动勾选同意
3. 说明必要性
4. 加密存储
5. 访问审计
```

## 数据生命周期管理

### 收集阶段

```markdown
## 数据收集合规检查

### 必须满足
- [ ] 明确告知收集目的
- [ ] 说明收集的信息类型
- [ ] 获得用户同意（勾选/点击）
- [ ] 敏感信息单独授权
- [ ] 提供拒绝选项

### 收集方式规范
| 场景 | 规范做法 |
|------|---------|
| 注册时收集 | 显示隐私政策弹窗 |
| 功能触发收集 | 使用时即时授权 |
| 后台收集 | 启动时一次性告知 |
| SDK 收集 | 在隐私政策中披露 |
```

### 存储阶段

```markdown
## 数据存储合规检查

### 存储位置
- [ ] 境内存储（涉及敏感数据）
- [ ] 服务器有安全认证
- [ ] 备份策略完善

### 存储安全
- [ ] 敏感数据加密存储
- [ ] 数据库访问控制
- [ ] 定期安全审计
- [ ] 日志记录完整

### 存储期限
| 数据类型 | 保存期限 | 到期处理 |
|---------|---------|---------|
| 账号信息 | 账号存续期间 | 注销后删除 |
| 交易记录 | 法定保存期 | 按规定处理 |
| 日志数据 | 6个月以上 | 定期清理 |
```

### 使用阶段

```markdown
## 数据使用合规检查

### 使用限制
- [ ] 在授权范围内使用
- [ ] 不超出告知目的
- [ ] 新用途需重新授权

### 共享限制
- [ ] 共享前告知用户
- [ ] 签署数据处理协议
- [ ] 最小范围共享
- [ ] 接收方有保护措施
```

### 删除阶段

```markdown
## 数据删除合规检查

### 删除触发条件
- 用户主动注销账号
- 用户申请删除数据
- 数据保存期限到期
- 业务终止

### 删除要求
- [ ] 主库数据删除或匿名化
- [ ] 备份数据定期清理
- [ ] 日志中个人信息脱敏
- [ ] 通知第三方同步删除
```

## 数据出境

### 何时需要安全评估

| 场景 | 是否需要评估 |
|------|------------|
| 数据存境外服务器 | 需要 |
| 向境外传输数据 | 需要 |
| 境外访问境内数据 | 视情况 |
| 境内存储境内访问 | 不需要 |

### 合规路径

```markdown
## 数据出境合规路径

### 方式一：安全评估（强制）
适用：关键信息基础设施运营者、处理100万人以上数据

### 方式二：标准合同
适用：一般数据处理者

### 方式三：个人信息保护认证
适用：集团内部跨境传输

### 评估流程
1. 自评估
2. 签署合同/申报材料
3. 提交网信部门
4. 等待审批（一般45个工作日）
```

## 等级保护

### 定级标准

| 等级 | 典型系统 | 主要要求 |
|------|---------|---------|
| 一级 | 内部办公 | 自主保护 |
| 二级 | 一般业务 | 指导保护（需备案）|
| 三级 | 重要业务、涉及大量个人信息 | 监督保护（需评测）|
| 四级 | 涉及国计民生 | 强制保护 |
| 五级 | 国家安全 | 专门保护 |

### 三级等保核心要求

```markdown
## 等保三级核心要求

### 技术要求
- [ ] 安全区域边界（防火墙）
- [ ] 入侵检测系统
- [ ] 漏洞扫描
- [ ] 安全审计
- [ ] 数据加密
- [ ] 备份恢复

### 管理要求
- [ ] 安全管理制度
- [ ] 安全管理机构
- [ ] 人员安全管理
- [ ] 系统建设管理
- [ ] 运维管理

### 评测周期
- 首次：系统上线前
- 复评：每年一次
```

## 违规处罚

| 违规行为 | 处罚 |
|---------|------|
| 未经同意收集个人信息 | 警告 → 罚款 → 停业整顿 |
| 过度收集 | 责令改正 → 罚款 |
| 数据泄露 | 罚款 + 赔偿 + 刑责 |
| 未做等保 | 罚款 + 责令整改 |
| 数据违规出境 | 罚款 + 停止出境 |

**处罚上限**：5000万元或上年度营业额5%
