# 硬件/IoT 验证

## 推荐实验

- **功能样机验证**：核心功能实现与稳定性测试
- **用户体验测试**：可用性、易用性与满意度评估
- **成本与良率试算**：BOM成本分析与量产良率预估
- **安全/合规预评估**：认证要求与合规可行性分析
- **众筹/预售测试**：市场需求验证与用户反馈收集

## 证据门槛（示例）

- 关键功能可稳定实现（样机测试通过率 >= 90%）
- BOM成本满足目标价位（毛利空间 >= 30%）
- 用户满意度 >= 4.0/5.0
- 预售转化率 >= 5%（早期需求验证）
- 主要认证路径明确且可行

## 决策阈值

- **继续推进**：功能可行、成本可控、用户需求明确、合规风险可控
- **需要优化**：一项关键指标未达标，识别改进点（设计、材料、供应链）后重新测试
- **重新构思**：多项关键指标未达标，或成本/合规风险过高

## 实验设计框架

### 1. 技术可行性验证
- **方法**：功能样机开发，核心功能测试
- **样本**：2-5台样机，多场景测试
- **周期**：4-8周样机开发测试期
- **指标**：功能实现度、性能指标达标率、稳定性（故障间隔时间）、功耗表现

### 2. 用户体验验证
- **方法**：可用性测试，用户交互观察
- **样本**：15-20名目标用户，分场景测试
- **周期**：1-2周用户测试期
- **指标**：任务完成率、学习曲线、操作错误率、满意度评分、改进建议

### 3. 成本与良率验证
- **方法**：BOM成本分析，小批量试产
- **样本**：10-50台试产，供应链评估
- **周期**：2-4周成本分析期
- **指标**：BOM成本、组装成本、测试良率、预期量产良率、目标毛利达成度

### 4. 合规认证验证
- **方法**：认证要求分析，第三方预评估
- **样本**：目标市场认证标准，认证机构咨询
- **周期**：2-3周合规研究期
- **指标**：认证清单完整性、测试通过预估、认证周期预估、认证成本预估

### 5. 市场需求验证
- **方法**：众筹/预售测试，早期用户反馈
- **样本**：至少500名潜在购买者触达
- **周期**：2-4周预售测试期
- **指标**：预售转化率、预售金额、目标用户画像、价格敏感度、功能优先级反馈
