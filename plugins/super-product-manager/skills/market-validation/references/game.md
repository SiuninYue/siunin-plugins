# 游戏/娱乐产品验证

## 推荐实验

- **核心玩法原型测试**：完成率与趣味性评分
- **小规模封测**：次日/7日/30日留存率
- **付费功能测试**：转化率与平均付费金额
- **美术风格偏好测试**：A/B测试不同视觉风格

## 证据门槛（示例）

- 核心玩法完成率 >= 70%
- 次日留存 >= 40%，7日留存 >= 20%
- 付费转化率 >= 3%（测试期间）
- 美术风格偏好度 >= 60%（A/B测试）

## 决策阈值

- **继续推进**：留存和付费指标达到门槛，且玩法反馈积极
- **需要优化**：一项关键指标未达标，识别改进点后重新测试
- **重新构思**：多项指标未达标或核心玩法反馈负面

## 实验设计框架

### 1. 玩法核心验证
- **方法**：可玩原型测试，核心循环体验
- **样本**：30-50名目标玩家，分经验水平
- **周期**：1-2小时游戏体验，即时反馈
- **指标**：任务完成率、游戏时长、趣味性评分、重玩意愿

### 2. 留存验证
- **方法**：小规模封测，追踪玩家行为
- **样本**：100-500名玩家，分渠道获取
- **周期**：至少30天，观察完整留存曲线
- **指标**：次日、7日、30日留存率，日均游戏时长，session频率

### 3. 商业化验证
- **方法**：有限付费功能测试，价格敏感度调研
- **样本**：封测玩家中的付费测试组
- **周期**：与封测同期进行
- **指标**：付费转化率、平均付费金额、付费时点、付费动机

### 4. 技术性能验证
- **方法**：压力测试、多设备兼容性测试
- **样本**：不同配置设备，模拟高并发场景
- **周期**：1-2周技术测试期
- **指标**：帧率稳定性、加载时间、崩溃率、内存占用

### 5. 市场定位验证
- **方法**：竞品对比测试，目标玩家群体调研
- **样本**：竞品玩家和潜在新玩家
- **周期**：2-3周对比分析期
- **指标**：差异化认可度、切换意愿、推荐可能性
