# 平台/双边验证

## 推荐实验

- **单边供给冷启动测试**：供给侧获取成本与留存率
- **需求侧转化测试**：用户活跃度与付费意愿
- **撮合成功率验证**：匹配效率与满意度追踪
- **网络效应测试**：用户增长与平台价值关系

## 证据门槛（示例）

- 供给侧留存 >= 30%（月留存）
- 撮合成功率 >= 20%（有效交易/互动）
- 需求侧周活跃率 >= 25%
- 网络效应指标：用户价值随规模增长（R² > 0.5）

## 决策阈值

- **继续推进**：供给留存和撮合成功率均达到门槛，且网络效应显现
- **需要优化**：一项指标未达标，识别改进点（获客策略、匹配算法、激励机制）后重新测试
- **重新构思**：两项指标均未达标，或双边市场冷启动困难无法突破

## 实验设计框架

### 1. 供给侧验证
- **方法**：供给方获取成本测试，留存率追踪
- **样本**：至少50名供给方，分渠道获取
- **周期**：2-4周供给测试期
- **指标**：获客成本、留存率、活跃度、内容/服务产出质量、收益满意度

### 2. 需求侧验证
- **方法**：需求方转化测试，用户行为追踪
- **样本**：至少100名需求方，分场景测试
- **周期**：2-4周需求测试期
- **指标**：注册转化率、活跃率、使用频率、满意度、付费意愿

### 3. 撮合效率验证
- **方法**：匹配算法测试，交易成功率追踪
- **样本**：同时有供给和需求的测试场景
- **周期**：2-3周撮合测试期
- **指标**：匹配成功率、响应时间、交易完成率、双方满意度、纠纷率

### 4. 网络效应验证
- **方法**：控制用户增长节奏，观察平台价值变化
- **样本**：分批次引入用户，追踪网络形成
- **周期**：4-8周网络建设期
- **指标**：用户间连接数、交易频率增长、单位用户价值变化、平台活跃度与规模关系

### 5. 冷启动策略验证
- **方法**：最小可行平台测试，人工辅助撮合
- **样本**：小规模双边用户，人工运营介入
- **周期**：2-4周冷启动测试期
- **指标**：冷启动成功率、人工介入成本、自运转达成时间、早期用户满意度
