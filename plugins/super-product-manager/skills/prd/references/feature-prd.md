## 模板使用说明

### 什么时候使用这个模板

**功能PRD适用于**：
- 新增用户功能
- 改造现有功能
- 用户交互相关的需求

**不适用的场景**：
- 纯数据埋点需求 → 使用数据PRD
- 纯API对接需求 → 使用API PRD

### 使用技巧

1. **先写用户故事**：确保从用户视角出发
2. **验收标准要可测试**：避免"提升体验"这类模糊描述
3. **明确边界**：In/Scope要清晰，避免范围蔓延
4. **数据驱动**：每个功能都要有对应的指标
5. **考虑风险**：技术依赖、业务依赖要提前识别
