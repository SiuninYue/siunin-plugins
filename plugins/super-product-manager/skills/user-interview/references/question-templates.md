# 访谈问题模板库

## 问题设计框架

访谈问题分为 5 个阶段，逐步深入：

```
背景问题 → 问题探索 → 行为挖掘 → 需求验证 → 收尾问题
```

---

## 1. 背景问题

**目的**：了解用户是谁、做什么、建立信任

```markdown
## 背景问题模板

- 你的工作/角色是什么？
- 你平时一天是怎么过的？
- 你主要负责什么？
- 你在这个领域工作多久了？
- 你们团队有多少人？
```

**注意**：背景问题不要问太多（2-3 分钟），快速进入正题。

---

## 2. 问题探索

**目的**：确认问题是否存在、多严重

```markdown
## 问题探索模板

### 确认问题存在
- 你最近一次遇到 [问题] 是什么时候？
- 当时的情况是怎样的？能具体描述一下吗？
- 这个问题多久出现一次？

### 评估问题严重性
- 这个问题对你的影响有多大？(1-10分)
- 因为这个问题，你损失过什么？（时间/金钱/机会）
- 如果这个问题一直不解决，会怎样？

### 深挖问题细节
- 能给我讲一个具体的例子吗？
- 当时你的感受是什么？
- 这个问题最让你头疼的是什么？
```

---

## 3. 行为挖掘

**目的**：了解现有解决方案和付费意愿

```markdown
## 行为挖掘模板

### 现有解决方案
- 你现在是怎么解决这个问题的？
- 用什么工具/方法？
- 为什么选择这个方案？
- 这个方法有什么不满意的地方？

### 付费意愿
- 你为这个问题花过钱吗？花了多少？
- 你目前在用什么付费产品/服务？
- 你愿意花多少钱来解决这个问题？
- 什么情况下你会考虑换一个方案？

### 替代方案
- 你试过其他方案吗？
- 为什么放弃了？
- 如果没有现在的方案，你会怎么办？
```

---

## 4. 需求验证

**目的**：验证解决方案方向（谨慎使用）

```markdown
## 需求验证模板

### 功能优先级
- 如果有个产品能 [你的解决方案]，你觉得最重要的是什么？
- 什么功能是你完全不需要的？
- 什么功能是"有了更好，没有也行"？

### 付费意愿（具体）
- 什么情况下你会付费？
- 你能接受的价格范围是？
- 你更喜欢一次性付费还是订阅？

### 使用场景
- 你会在什么情况下使用这个产品？
- 多久会用一次？
- 和你现在的工具怎么配合？
```

**警告**：需求验证问题容易得到假信息，要结合前面的行为数据综合判断。

---

## 5. 收尾问题

**目的**：获取更多资源、保持联系

```markdown
## 收尾问题模板

### 补充信息
- 还有什么我没问到但你想说的？
- 关于这个问题，你觉得我应该知道什么？

### 拓展资源
- 你身边还有谁可能也有这个问题？能介绍认识吗？
- 有什么社群/论坛是讨论这个问题的吗？

### 后续联系
- 如果我们做出产品原型，方便找你测试吗？
- 最方便的联系方式是什么？
```

## 场景专用问题库

### 验证新产品想法

```markdown
1. 你上次遇到 [问题] 是什么时候？具体情况是？
2. 你当时是怎么解决的？
3. 这个方案有什么不满意的地方？
4. 你为解决这个问题花过钱吗？
5. 如果有更好的方案，你愿意花多少钱？
```

### 验证功能优先级

```markdown
1. 你现在使用 [产品] 最常用的功能是什么？
2. 有什么功能你很少用或者从来不用？
3. 你希望增加什么功能？为什么？
4. 如果只能保留 3 个功能，你会选哪些？
5. 缺少什么功能让你考虑换产品？
```

### 了解竞品体验

```markdown
1. 你用过哪些类似的产品？
2. 你现在主要用哪个？为什么？
3. 之前用的产品为什么放弃了？
4. 你觉得现在用的产品最大的问题是什么？
5. 你会向朋友推荐这个产品吗？为什么？
```

### 了解付费决策

```markdown
1. 你是怎么发现这个产品的？
2. 决定付费前考虑了多久？
3. 最后是什么让你决定付费的？
4. 付费后有什么不满意的地方吗？
5. 你会续费吗？为什么？
```

## 追问技巧

当用户给出模糊回答时，用这些问题追问：

| 用户说 | 追问 |
|-------|------|
| "挺好的" | "好在哪里？能具体说说吗？" |
| "可能会用" | "什么情况下会用？什么情况下不会用？" |
| "挺重要的" | "比 [X] 更重要还是没那么重要？" |
| "有时候" | "上一次是什么时候？多久一次？" |
| "应该会" | "你之前有没有因为类似原因买过什么？" |
