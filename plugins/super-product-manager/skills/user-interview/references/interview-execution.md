# 访谈执行指南

## 访谈前准备

### 准备清单

- [ ] 明确本次访谈的假设和目标
- [ ] 准备好问题大纲（但不要死板照读）
- [ ] 准备录音设备（征得同意）
- [ ] 准备笔记本
- [ ] 了解受访者基本背景
- [ ] 确认时间地点（线上确认链接可用）

### 假设记录表

```markdown
## 本次访谈要验证的假设

| 假设 | 验证方法 | 成功标准 |
|-----|---------|---------|
| [假设1] | 问 [什么问题] | [什么回答说明成立] |
| [假设2] | 问 [什么问题] | [什么回答说明成立] |
```

### 用户招募技巧

| 方法 | 优点 | 缺点 |
|-----|------|------|
| **付费招募平台** | 质量可控，效率高 | 成本高 |
| **目标用户社群** | 精准，真实 | 需要时间建立信任 |
| **现有用户** | 容易联系 | 可能有偏见 |
| **朋友转介绍** | 信任度高 | 样本有限 |

**不要做**：
- ❌ 找朋友家人（他们会说好话）
- ❌ 只找愿意配合的人（样本偏差）
- ❌ 没说明时长就约人

**要做**：
- ✅ 去目标用户聚集的地方
- ✅ 付费招募比免费更靠谱
- ✅ 说明大概时长和目的

## 访谈中执行

### 开场（2-3分钟）

```markdown
## 开场话术模板

"你好，感谢你抽出时间。我是 [名字]，正在研究 [领域]。
今天主要想了解你在 [场景] 方面的经历和看法。
访谈大概 [X] 分钟，你说的任何内容都会保密。
方便我录音吗？这样我可以专心听你说，不用一直记笔记。"
```

**开场目标**：
- 建立轻松氛围
- 说明目的和时长
- 获得录音许可

### 核心访谈技巧

| 技巧 | 说明 | 示例 |
|-----|------|------|
| **多听少说** | 你说话时间不超过 30% | 问完问题后等待 |
| **追问细节** | 不满足于笼统回答 | "能具体说说吗？" "然后呢？" |
| **沉默是金** | 他停顿时，等 3 秒再说话 | 沉默会让对方补充更多 |
| **不要引导** | 避免预设答案 | ❌ "你是不是觉得..." |
| **不要推销** | 你是来学习的，不是来卖东西的 | 不解释你的产品 |
| **记录原话** | 用户的原话比你的总结更有价值 | 直接记录引用 |

### 追问公式

```
用户说了一个观点
    ↓
"能举个具体例子吗？"
    ↓
用户给了一个例子
    ↓
"当时具体是什么情况？"
    ↓
用户描述了情况
    ↓
"那你是怎么处理的？"
    ↓
用户描述了行为
    ↓
"后来怎么样了？"
```

### 应对常见情况

| 情况 | 应对 |
|-----|------|
| 用户跑题 | "这个很有意思，我们一会儿再聊。先回到刚才的问题..." |
| 用户给出简短回答 | "能多说一点吗？""比如？" |
| 用户问你的看法 | "我很想听你的想法，我的看法我们最后再聊" |
| 用户说"没有问题" | "你上次使用的时候，有没有任何不顺畅的地方？" |
| 用户开始赞美你的想法 | 回到事实："谢谢！我更想知道你之前是怎么做的" |

### 收尾（2-3分钟）

```markdown
## 收尾话术模板

"时间差不多了，最后问几个问题：
- 还有什么我没问到但你想说的吗？
- 你身边还有谁可能也有这个经历？方便介绍认识吗？
- 如果后面有进展，我可以再联系你吗？

非常感谢你的时间，你说的对我帮助很大。"
```

## 远程访谈注意事项

### 工具准备

| 工具 | 推荐 |
|-----|------|
| 视频会议 | Zoom, 腾讯会议, 飞书 |
| 录屏 | OBS, 系统自带 |
| 笔记 | Notion, 飞书文档 |

### 远程访谈技巧

- **提前 5 分钟进入**，确认技术没问题
- **打开摄像头**，建立更好的连接
- **准备 B 方案**，网络不好时切换到电话
- **录制要双方同意**，开始前确认
- **控制时长**，线上注意力更容易分散
