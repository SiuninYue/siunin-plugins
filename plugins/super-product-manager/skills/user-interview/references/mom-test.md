# The Mom Test 方法论

## 为什么叫 The Mom Test？

你妈妈会说你的想法很棒。但她的鼓励对你一点用都没有。

**核心原则**：
1. 问过去的行为，不问未来的意愿
2. 问具体的事件，不问抽象的观点
3. 用户的行为比语言更真实

---

## 坏问题（得到假信息）

| 坏问题 | 问题出在哪 |
|-------|----------|
| 「你觉得这个想法怎么样？」 | 朋友会说好，敌人会说不好 |
| 「你会用这个产品吗？」 | 人人都说会，没人真的会 |
| 「你愿意付钱吗？」 | 口头承诺不算数 |
| 「你觉得 X 功能重要吗？」 | 都说重要，没人真的用 |
| 「你一般怎么做 X？」 | 得到的是「应该」不是「实际」 |

**为什么这些问题是坏的？**

- 它们问的是**意见**，而不是**事实**
- 它们问的是**未来**，而不是**过去**
- 它们让用户**预测行为**，而不是**描述行为**

---

## 好问题（得到真信息）

| 好问题 | 为什么好 |
|-------|---------|
| 「你上次遇到这个问题是什么时候？」 | 过去真实发生的事 |
| 「当时你怎么解决的？」 | 真实的行为 |
| 「你为这个花了多少钱/时间？」 | 付出 = 真实需求 |
| 「能给我演示一下吗？」 | 行动比语言真实 |
| 「你试过什么方案？为什么放弃了？」 | 真实的痛点 |

**好问题的特征**：

- 问的是**具体事件**，有时间、地点、细节
- 问的是**已经发生的行为**
- 问的是**真实付出**（时间、金钱、精力）

---

## 三个黄金法则

### 法则 1：谈论他们的生活，而不是你的想法

**错误做法**：
> "我们正在做一个能帮你管理任务的 App，你觉得怎么样？"

**正确做法**：
> "你平时怎么记录要做的事？用什么工具？"

### 法则 2：问具体的过去，而不是抽象的未来

**错误做法**：
> "如果有这个功能，你会用吗？"

**正确做法**：
> "你上次遇到这个问题时，花了多少时间解决？"

### 法则 3：少说多听

**目标**：你说话的时间不超过 30%

**技巧**：
- 问完问题后，等 3-5 秒再说话
- 用"然后呢？""具体说说？"引导对方多说
- 不要急于解释或推销

---

## 验证信号强度

| 信号强度 | 用户表现 | 可信度 |
|---------|---------|--------|
| **极强** | 主动付钱、主动推荐、持续使用 | ★★★★★ |
| **强** | 投入时间学习、愿意参与测试 | ★★★★ |
| **中** | 表示感兴趣、愿意再聊 | ★★★ |
| **弱** | 说"挺好的"、"有需要" | ★★ |
| **无效** | 说"我会用的"、"很有用" | ★ |

**记住**：口头承诺的价值接近于零。只有真实的行动才算数。

---

## 常见陷阱

### 陷阱 1：寻求认可

**症状**：你在寻找"验证"你想法的证据

**解药**：主动寻找"推翻"你想法的证据

### 陷阱 2：引导性提问

**坏例子**：
> "你是不是觉得现在的工具太复杂了？"

**好例子**：
> "你用这个工具的时候，遇到过什么困难吗？"

### 陷阱 3：只记住好的

**症状**：只记得支持你观点的反馈

**解药**：原样记录用户的话，事后再分析

### 陷阱 4：问错人

**症状**：问朋友、家人、同事

**解药**：找真正的目标用户，最好是陌生人
