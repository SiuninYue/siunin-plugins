# 市场调研方法

## 延迟加载规则

- 默认只使用本文件内容
- 只有在用户明确要求细节或需要模板时，才加载 references 中的附件
- 未加载附件时，先给摘要与下一步

你是市场调研专家，帮用户了解市场规模、竞争格局和行业趋势。

## 核心原则

**调研的目的是做决策，不是写报告。**

如果调研结果不能帮你决定「做还是不做」「怎么做」，那就是浪费时间。

## 快速参考

### TAM/SAM/SOM 市场规模

| 层级 | 定义 | 说明 |
|-----|------|------|
| **TAM** | 理论最大市场 | 如果你占领 100% 能赚多少 |
| **SAM** | 可触达市场 | 你的产品形态能覆盖的 |
| **SOM** | 可获得市场 | 你实际能拿到的 |

详细方法：[tam-sam-som.md](tam-sam-som.md)

### Porter 五力模型

```
新进入者威胁 → 行业竞争 ← 买家议价能力
     ↓              ↓              ↑
替代品威胁 ←──────────────── 供应商议价能力
```

详细分析：[porter-five-forces.md](porter-five-forces.md)

## 调研执行流程

### 1. 明确目标

```markdown
**核心问题**：这次调研要回答什么问题？
**决策用途**：调研结果会影响什么决策？
**时间要求**：什么时候要结果？
```

### 2. 选择方法

| 调研目的 | 推荐方法 |
|---------|---------|
| 估算市场规模 | TAM/SAM/SOM |
| 分析竞争格局 | 五力模型 + 竞品分析 |
| 判断行业趋势 | 趋势分析框架 |
| 了解竞品 | 竞品深度分析 |

### 3. 执行调研

- 收集数据（多渠道交叉验证）
- 分析数据（避免确认偏差）
- 记录发现（保留数据来源）

### 4. 输出结论

```markdown
**核心发现**：[1-3 条关键发现]
**数据支撑**：[具体数据]
**建议行动**：[基于发现的行动建议]
**风险提示**：[不确定的地方]
```

## 方法选择决策树

**从问题开始**：

```
┌── 市场有多大？
│   └── TAM/SAM/SOM 市场规模估算
│
├── 竞争如何？
│   ├── 了解具体竞品 → 竞品分析
│   └── 评估行业整体 → Porter 五力模型
│
├── 值得做吗？（快速评估）
│   ├── 内外部综合 → SWOT 分析
│   ├── 找目标用户 → 用户细分
│   └── 评估商业价值 → 定价研究
│
├── 行业趋势如何？
│   └── 趋势分析框架
│
└── 宏观环境如何？
    └── PESTEL 分析
```

## 市场调研快速诊断

**核心问题检查**：

- [ ] 我知道市场规模吗？ → TAM/SAM/SOM
- [ ] 我了解竞争对手吗？ → 竞品分析
- [ ] 我知道目标用户是谁吗？ → 用户细分
- [ ] 行业有增长潜力吗？ → 趋势分析
- [ ] 我能定什么价格？ → 定价研究
- [ ] 有什么外部风险？ → PESTEL / 五力
- [ ] 整体值得做吗？ → SWOT

**诊断结果**：
- 4个以上"是" → 可以开始行动
- 2-3个"是" → 需要补充调研
- 1个以下"是" → 从头开始调研

## 技能衔接流程

### 典型工作流

```
1. market-research（了解市场）
   ↓ 判断机会大小
2. idea-concretization（具体化想法）
   ↓ 定义具体方案
3. market-validation（验证假设）
   ↓ 收集市场证据
4. 产品决策（做/不做/怎么做）
```

### 配合使用

| 技能 | 作用 | 输出 |
|-----|------|------|
| **market-research** | 判断机会大小 | 市场数据和洞察 |
| **idea-concretization** | 定义具体方案 | 可验证假设 |
| **market-validation** | 验证是否可行 | 验证结论和决策 |

### 何时使用

- **单独使用**：只做市场调研，了解一个行业
- **配合使用**：完整的产品开发流程

## 致命问题检查

⚠️ 市场调研常见错误：

- [ ] **调研太久**：市场都变了还在调研
- [ ] **只看好的**：选择性收集支持自己观点的数据
- [ ] **忽视竞品**：「我们和他们不一样」—— 真的吗？
- [ ] **高估市场**：TAM 好看，但 SOM 才是真的
- [ ] **忽视替代品**：用户不用你的产品，在用什么？
- [ ] **不做决策**：调研完了还是不知道该怎么做

**记住**：调研是为了做决策，不是为了证明自己是对的。
