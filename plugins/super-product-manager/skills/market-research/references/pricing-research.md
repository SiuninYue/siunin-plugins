# 定价研究指南

## 定价的核心原则

**关键认知**：定价不是成本加成，而是价值交换

**核心问题**：用户愿意为你的价值付多少钱？

**定价三要素**：
1. **成本** - 你的底线
2. **价值** - 用户的上限
3. **竞争** - 市场的基准

```
价格区间：
成本底价 < 竞争基准 < 价值感知价 < 实际定价
```

---

## 定价方法

### 方法一：成本加成定价

**原理**：成本 + 目标利润 = 价格

```
价格 = 单位成本 × (1 + 目标利润率)
```

**示例**：
```markdown
单位成本 = ¥50
目标利润率 = 60%
价格 = ¥50 × 1.6 = ¥80
```

**优点**：简单、确保盈利
**缺点**：忽视用户价值和竞争
**适用**：标准化产品、成本稳定

---

### 方法二：价值导向定价

**原理**：基于用户获得的价值定价

```
价格 = 用户获得价值 × 愿意支付比例
```

**示例**：
```markdown
用户时间节省 = ¥1000/月
效率提升 = ¥500/月
总价值 = ¥1500/月
愿意支付比例 = 20%
价格 = ¥1500 × 20% = ¥300/月
```

**优点**：最大化收益、反映真实价值
**缺点**：价值量化难、用户教育成本高
**适用**：高价值差异化产品

---

### 方法三：竞争导向定价

**原理**：参考竞争对手定价

```
价格 = 竞品价格 × 调整系数
```

**示例**：
```markdown
主要竞品价格 = ¥100/月
我们功能更好，调整系数 = 1.2
价格 = ¥100 × 1.2 = ¥120/月
```

**优点**：市场接受度高
**缺点**：陷入价格战、忽视差异化
**适用**：同质化产品

---

### 方法四：动态定价

**原理**：根据供需实时调整价格

**示例场景**：
- 早鸟优惠（早期用户低价）
- 限时促销（库存清空）
- 差异定价（不同用户不同价）
- 拥堵定价（高峰时段涨价）

**优点**：最大化收益
**缺点**：复杂、用户可能不满
**适用**：平台、电商、订阅

## 价格敏感度测试

### Van Westendorp 模型

**原理**：通过四个问题找到最优价格区间

**四个问题**：
1. **什么价格太贵，不考虑买？**
2. **什么价格有点贵，但还会考虑？**
3. **什么价格很便宜，怀疑质量？**
4. **什么价格超值，不用想就买？**

**结果分析**：
```
价格
 │
 │     可接受价格区间
 │    ┌─────────────┐
 ├──┤             ├── 最优价格点
 │  └─────────────┘
 │
 └───────────────────
```

**解读**：
- 太贵线和有点贵线交点 = 价格上限
- 超值线和便宜线交点 = 价格下限
- 两条中间曲线交点 = 最优价格

---

### Gabor-Granger 法

**原理**：直接询问是否购买，逐步调整价格

**流程**：
```
1. 告知一个价格，问是否购买
2. 如果是 → 提高价格，再问
3. 如果否 → 降低价格，再问
4. 重复到找到转折点
```

**输出**：
- 价格-需求曲线
- 最优价格点（收益最大化）

---

### A/B 测试定价

**方法**：同时测试多个价格，比较转化率

**示例**：
```markdown
A组：¥99/月 → 转化率 3%
B组：¥129/月 → 转化率 2.5%
C组：¥159/月 → 转化率 2%

收益计算：
A组：¥99 × 3% = ¥2.97/访客
B组：¥129 × 2.5% = ¥3.23/访客 ✅
C组：¥159 × 2% = ¥3.18/访客

最优：B组 ¥129/月
```

|-----|---------|------|------|

## 定价策略模板

### 免费增值 (Freemium)

**结构**：
```markdown
┌─────────────┬─────────────┬─────────────┐
│   免费版    │   专业版    │   企业版    │
│   ¥0/月     │   ¥99/月    │   ¥499/月   │
├─────────────┼─────────────┼─────────────┤
│ 基础功能    │ 全部功能    │ 全部功能    │
│ 使用限额    │ 无限额      │ 无限额      │
│ 社区支持    │ 优先支持    │ 专属支持    │
│            │ 去广告      │ 去广告      │
│            │            │ SLA保证     │
└─────────────┴─────────────┴─────────────┘
```

**转化目标**：免费用户 → 付费用户

**关键指标**：
- 免费用户转付费率
- 付费用户留存率
- ARPU（每用户平均收入）

---

### 分层定价 (Tiered Pricing)

**设计原则**：
- 层级分明（3-4个）
- 差异明显（功能/限制）
- 价格锚定（中间最受欢迎）

**示例**：
```markdown
| 层级 | 价格 | 目标用户 |
|-----|------|---------|
| 个人版 | ¥29/月 | 个人用户 |
| 团队版 | ¥99/月 | 小团队 |
| 企业版 | ¥399/月 | 大企业 |
```

---

### 使用量定价 (Usage-based)

**原理**：按实际使用量计费

**示例**：
```markdown
API调用计费：
- 0-1万次：免费
- 1万-10万次：¥0.01/次
- 10万次以上：¥0.005/次

云存储计费：
- 0-10GB：免费
- 10GB-1TB：¥0.1/GB/月
- 1TB以上：¥0.05/GB/月
```

**优点**：用多少付多少、降低使用门槛
**缺点**：收入不稳定、用户担心成本失控

---

### 企业定制 (Enterprise)

**定价因素**：
```markdown
基础价格：¥X/年
+
用户数：¥X/用户/年
+
功能模块：¥X/模块
+
实施服务：¥X
+
培训服务：¥X
=
企业总价：¥X
```

## 定价心理学

### 心理价格效应

| 效应 | 原理 | 应用 |
|-----|------|------|
| **尾数定价** | ¥99比¥100感觉便宜很多 | 定价¥99而不是¥100 |
| **锚定效应** | 第一个价格影响判断 | 先展示高价再打折 |
| **诱饵效应** | 加入一个差选项突显好选项 | 三个套餐中推中间的 |
| **损失厌恶** | 不买比买贵更难受 | 强调限时/限量 |
| **价格框架** | 表述方式影响感知 | "每天¥3"比"每月¥99" |

### 定价呈现技巧

```markdown
## 定价呈现优化

### 分解价格
原价：¥1200/年
优化后：每天只要 ¥3

### 对比价格
原价：¥199/月
优化后：¥199/月（相当于每天一杯咖啡）

### 强调价值
原价：¥99/月
优化后：节省10小时/月 × ¥50/小时 = 节省¥500/月，只需¥99

### 社会证明
"已有10,000+用户选择"
```

## 常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **只看成本** | 成本低就卖便宜 | 看用户价值 |
| **定价后不调整** | 市场在变 | 定期复盘 |
| **所有人一个价** | 放弃差异化收益 | 细分定价 |
| **忽视心理价格** | ¥100比¥99感觉贵很多 | 用尾数定价 |
| **频繁调价** | 用户不信任 | 谨慎调整 |
| **没有涨价计划** | 通胀和成本上升 | 设计涨价路径 |
| **价格战** | 大家都输 | 差异化 |
