# 用户细分指南

## 什么是用户细分

**核心问题**：你的产品到底卖给谁？

**定义**：把大市场分成小群体，找到最有价值的细分市场

**目的**：
- 聚焦目标用户
- 提高营销效率
- 优化产品设计
- 提升转化率

## B2C 用户细分

### 人口统计学细分

| 维度 | 细分选项 | 应用示例 |
|-----|---------|---------|
| **年龄** | Z世代/ Millennials/ X世代/ 银发族 | 产品风格、营销语言 |
| **性别** | 男/女/其他 | 颜色、功能偏好 |
| **收入** | 低/中/高收入 | 定价策略 |
| **地域** | 一线/新一线/二线/三四线 | 渠道、推广策略 |
| **教育** | 高中/本科/硕士+ | 内容复杂度 |
| **家庭** | 单身/已婚/有孩/无孩 | 需求场景 |

### 行为细分

| 维度 | 细分选项 | 应用示例 |
|-----|---------|---------|
| **使用频次** | 重度/中度/轻度/非用户 | 运营策略 |
| **使用时长** | 高时长/中时长/低时长 | 产品设计 |
| **购买历史** | 首购/复购/流失 | 促活策略 |
| **功能偏好** | 功能A型/功能B型 | 功能优先级 |
| **价格敏感** | 价格敏感/价格不敏感 | 定价和促销 |

### 心理细分

| 维度 | 细分选项 | 应用示例 |
|-----|---------|---------|
| **价值观** | 环保/性价比/品质/地位 | 品牌定位 |
| **生活方式** | 忙碌/休闲/健康/社交 | 场景营销 |
| **兴趣爱好** | 运动/科技/美食/旅行 | 内容策略 |
| **个性特质** | 保守/开放/从众/个性 | 产品风格 |

### 场景细分

| 维度 | 细分选项 | 应用示例 |
|-----|---------|---------|
| **时间** | 早晨/通勤/午休/晚上/深夜 | 推送时机 |
| **地点** | 家/办公室/户外/通勤路上 | 渠道策略 |
| **情境** | 紧急/日常/特殊场合 | 功能设计 |
| **任务** | 工作/学习/娱乐/社交 | 产品定位 |

## B2B 用户细分

### 企业特征细分

| 维度 | 细分选项 | 应用示例 |
|-----|---------|---------|
| **行业** | 制造/金融/医疗/教育/零售 | 解决方案设计 |
| **规模** | 微型/小型/中型/大型 | 定价和销售策略 |
| **阶段** | 初创/成长/成熟 | 产品和服务类型 |
| **地域** | 国内/海外/具体区域 | 团队部署 |

### 决策者细分

| 角色 | 决策权重 | 关注点 |
|-----|---------|-------|
| **CEO/创始人** | 高 | 战略匹配、ROI |
| **部门负责人** | 高 | 功能匹配、效率提升 |
| **技术负责人** | 中 | 技术兼容、安全性 |
| **采购/财务** | 中 | 价格、合同条款 |
| **最终用户** | 低 | 易用性、体验 |

### 技术成熟度细分

| 类型 | 特征 | 策略 |
|-----|------|------|
| **技术领先** | 乐于尝试新技术 | 强调创新、前沿 |
| **技术跟随** | 等技术成熟再采用 | 强调稳定、案例 |
| **技术保守** | 很少采用新技术 | 强调安全、兼容 |

## 目标细分选择

### 评估矩阵

```
高市场吸引力
    │
    │      [细分A]
    │      重点进入
    │
    │  [细分B]     [细分C]
    │  谨慎进入     不考虑
    │
    │  [细分D]
    │  伺机进入
    └───────────────────── 低进入可行性
         低进入可行性
```

| 象限 | 策略 | 说明 |
|-----|------|------|
| **高吸引力/高可行性** | 重点进入 | 最优目标细分 |
| **高吸引力/低可行性** | 谨慎进入 | 评估是否提升能力 |
| **低吸引力/高可行性** | 伺机进入 | 作为备选或补充 |
| **低吸引力/低可行性** | 不考虑 | 放弃 |

## 细分验证

### 验证方法

| 方法 | 说明 | 成本 |
|-----|------|------|
| **数据分析** | 分析现有用户数据 | 低 |
| **用户访谈** | 深度访谈细分用户 | 中 |
| **问卷调研** | 大样本问卷验证 | 中 |
| **A/B测试** | 测试不同细分响应 | 中 |
| **小规模投放** | 针对性投放测试 | 高 |

### 验证清单

```markdown
## 细分市场验证

### 假设验证
- [ ] 该细分真的存在吗？
- [ ] 该细分有付费意愿吗？
- [ ] 该细分规模足够大吗？
- [ ] 我们能触达该细分吗？
- [ ] 该细分增长有潜力吗？

### 数据验证
- [ ] 有多少用户属于该细分？
- [ ] 该细分的转化率如何？
- [ ] 该细分的留存率如何？
- [ ] 该细分的 LTV 是多少？
- [ ] 该细分的 CAC 是多少？

### 竞争验证
- [ ] 竞争对手在该细分的表现？
- [ ] 我们在该细分的差异化？
- [ ] 该细分的进入壁垒？
```

## 细分策略

### 聚焦策略

| 策略 | 适用情况 | 风险 |
|-----|---------|------|
| **单一细分聚焦** | 资源有限，细分够大 | 把鸡蛋放一个篮子 |
| **少数细分聚焦** | 有多个优质细分 | 资源分散 |
| **多细分覆盖** | 市场成熟，资源充足 | 失去焦点 |

## 常见错误

| 错误 | 问题 | 正确做法 |
|-----|------|---------|
| **细分太粗** | 细分后差异不大 | 细分到有明显差异 |
| **细分太细** | 细分太小不值得做 | 评估细分规模 |
| **凭感觉细分** | 没有数据支撑 | 用数据验证 |
| **只看人口统计学** | 忽视行为和心理 | 多维度综合 |
| **细分后不聚焦** | 哪个都想做 | 选择1-2个重点 |
| **一成不变** | 市场在变 | 定期复盘细分 |

## 快速检查清单

做用户细分前，确保：

- [ ] 明确细分目的（营销/产品/销售）
- [ ] 收集用户数据（现有/潜在）
- [ ] 选择细分维度
- [ ] 设计细分方案

细分后，确保：

- [ ] 每个细分有明确画像
- [ ] 评估每个细分价值
- [ ] 选择目标细分
- [ ] 验证细分假设
- [ ] 制定进入策略
- [ ] 持续追踪细分表现
