# Progress Manager 深度模块化拆分 (F18)

- **Change ID**: 20260521-pm-modularize-7019
- **Date**: 2026-05-21
- **Component**: progress_manager.py
- **Root Cause**: 13,587 行单文件导致 AI session 无法全量读取，影响高风险修复定位

## Changes

将 `hooks/scripts/progress_manager.py` 按职责拆分为 10 个独立子模块，采用 Method A（零测试改动）：

| 模块 | 职责 | 行数 |
|------|------|------|
| `lock_manager.py` | 文件锁与事务 | 134 |
| `state_io.py` | Schema 规范化 | 210 |
| `git_utils.py` | Git 基础操作 | 788 |
| `git_context.py` | Runtime / execution context | 352 |
| `worktree_handler.py` | Worktree 路径解析 | 266 |
| `route_sync.py` | Active routes 同步 | 667 |
| `route_commands.py` | 路由 CLI 命令 | 849 |
| `route_preflight.py` | Route ownership preflight | 286 |
| `evaluator_gateway.py` | Evaluator 调用编排 | 193 |
| `pm_runtime.py` | CLI `__main__` / import module resolver | 32 |

主入口 `progress_manager.py` 从 13,587 行降至 10,452 行（减少 23%），保留 dispatcher + compatibility re-export 角色。

## Design Decisions

- **P0 硬规则**: 16 个被测试 patch 的函数逐个应用 Case A/B/C 决策树，确保 `patch("pm.X")` 路径在提取后继续生效
- **不可提取组**: `analyze_git_auto_preflight`、`_resolve_main_repo_path`、`_save_progress_payload_at_root` 保留在 pm.py（Case C：直接测试 + 内部调用 mocked dep）
- **CLI scope 兼容**: `pm_runtime.py` 让拆分模块优先绑定 CLI 运行中的 `__main__`，避免 `python progress_manager.py --project-root ...` 时二次 import 丢失 scope
- **循环依赖**: evaluator_gateway.py 通过运行时 resolver 延迟绑定 pm.py，打破与 pm.py 的循环
