# Claude-Plugins 项目规则

## 插件/Skill 编辑工作流

编辑插件或 skill 相关文件时，**必须使用以下方式**：

1. **plugin-dev:skill-development** - 创建/编辑 skill
2. **plugin-dev:agent-creator** - 创建/编辑 agent
3. **plugin-dev:command-development** - 创建/编辑 command

## 使用方法

使用 Skill 工具：

```
Skill tool:
  skill: "plugin-dev:skill-development"
  args: "创建/编辑 skill 的说明"
```

或使用 Task 工具调用 agent：

```
Task tool:
  subagent_type: "plugin-dev:agent-creator"
  prompt: "创建/编辑 agent 的说明"
```

## 为什么需要这样做？

- 确保符合 frontmatter 规范（name, description, version, scope 等）
- 确保使用正确的 writing style（imperative/infinitive form, third-person）
- 确保遵循 progressive disclosure 原则
- 自动更新相关文档和 marketplace/plugin.json

## `.claude` 文件同步规则（主线状态 vs 本地快照）

- **应合并到主线（项目状态）**：`.claude/progress.json`、`.claude/progress.md`、`.claude/project_memory.json`
- **保留为项目规则文档**：`.claude/.local.md`
- **不应作为功能 PR 内容（本地运行快照）**：`.claude/checkpoints.json`、`.claude/settings.local.json`
